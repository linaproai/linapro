## ADDED Requirements

### Requirement: 列表搜索区展开折叠按钮必须仅在存在可折叠内容时显示

默认管理工作台 SHALL 在列表页搜索区根据当前实际布局决定是否显示展开/折叠按钮。若搜索条件和搜索操作在当前断点下不需要折叠即可完整展示，系统 MUST 不显示展开/折叠按钮；若存在超出折叠保留行数的搜索条件，系统 SHALL 显示展开/折叠按钮并保持原有折叠交互。

#### Scenario: 一行搜索区隐藏展开折叠按钮

- **WHEN** 用户打开一个默认管理工作台列表页
- **AND** 该列表页搜索条件在当前视口下实际只占一行
- **THEN** 搜索区直接显示搜索条件、重置按钮和搜索按钮
- **AND** 搜索区不显示展开或折叠按钮

#### Scenario: 多行搜索区保留展开折叠按钮

- **WHEN** 用户打开一个默认管理工作台列表页
- **AND** 该列表页搜索条件在当前视口下存在超出折叠保留行数的内容
- **THEN** 搜索区显示展开或折叠按钮
- **AND** 用户可以通过该按钮展开或收起额外搜索条件

#### Scenario: 搜索区断点变化后重新判断按钮可见性

- **WHEN** 用户调整浏览器视口或系统切换到会改变搜索区布局的断点
- **THEN** 系统重新判断搜索区是否存在可折叠内容
- **AND** 展开或折叠按钮的可见性与调整后的实际搜索区行数保持一致
