# 修复角色权限树模式切换丢失原始选择

## 背景

社区`Issue #82`反馈角色编辑抽屉中权限树在“独立选择”和“父子联动”之间切换后，会让管理员无法确认原始选择的按钮权限，并可能在保存时把权限扩大为父级菜单或同级节点。该问题影响角色授权关系维护，属于默认用户权限模块的前端交互缺陷。

## 范围

- 修复角色编辑抽屉中菜单权限树的模式切换状态同步。
- 保证仅选择按钮权限的角色在编辑、切换模式和保存后，不会被前端提交为额外父级或同级菜单权限。
- 补充覆盖该反馈场景的自动化测试。

## 非目标

- 不修改`apps/lina-core`角色、菜单或权限接口契约。
- 不调整`/api/v1/menus/all`的路由菜单补齐设计。
- 不修改数据库结构、Seed 数据或角色权限存储模型。
