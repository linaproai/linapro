## ADDED Requirements

### Requirement: SQL 启动命令不得依赖驱动多语句执行能力
系统 SHALL 在宿主 `init` 与 `mock` 命令执行 SQL 文件时，将每个文件解析为有序的独立 SQL 语句并逐条执行，而不是依赖数据库连接串中的驱动级 `multiStatements` 能力；任一语句执行失败时，系统 MUST 立即停止当前文件剩余语句及后续文件的执行，并向调用方返回失败状态。

#### Scenario: 多语句 SQL 文件按顺序逐条执行
- **WHEN** `init` 或 `mock` 读取到一个包含多条 SQL 语句的目标文件
- **THEN** 系统按文件内语句出现顺序逐条执行这些语句
- **AND** 空白片段与纯注释片段不会被当作可执行语句

#### Scenario: 语句执行失败后立即中止
- **WHEN** `init` 或 `mock` 在执行某个 SQL 文件的中间语句时收到数据库错误
- **THEN** 系统立即停止当前文件剩余语句与后续 SQL 文件的执行
- **AND** 命令返回失败状态
- **AND** 错误信息继续包含失败文件名，便于定位问题
