## ADDED Requirements

### Requirement: 动态插件授权审查窗口展示路由暴露面清单

系统 SHALL 在动态插件安装或启用前的授权审查窗口中，额外展示当前 release 声明的动态路由清单，用于帮助管理员审查该插件将新增的对外访问入口。该清单 MUST 作为独立于 `hostServices` 授权范围的并列信息区展示，不得被混入宿主服务授权项语义中；在视觉层级上，宿主服务授权范围 MUST 先于路由信息展示。

#### Scenario: 安装前展示动态路由清单
- **WHEN** 宿主准备安装一个声明了动态路由合同的动态插件，并打开安装审查窗口
- **THEN** 审查窗口展示该插件当前 release 的动态路由列表
- **AND** 每条路由至少展示 HTTP 方法、宿主真实公开路径、访问级别、权限标识和摘要等关键信息
- **AND** 该列表作为独立的路由信息区展示，而不是混入宿主服务授权卡片或资源列表中
- **AND** 宿主服务授权范围先展示，路由信息区后展示，确保授权信息优先可见

#### Scenario: 启用前展示待生效 release 的动态路由清单
- **WHEN** 一个尚未形成最终启用态的动态插件 release 在启用前需要打开授权审查窗口，且该 release 声明了动态路由合同
- **THEN** 宿主展示该 release 当前声明的动态路由清单，而不是历史 release 的路由数据
- **AND** 管理员可以在确认启用前审查本次即将对外生效的路由暴露面

#### Scenario: 路由较多时默认折叠后续条目
- **WHEN** 动态插件安装或启用前的授权审查窗口打开，且当前 release 声明的动态路由数量大于两条
- **THEN** 路由信息区默认仅展示前两条路由
- **AND** 宿主提供“展开”操作，让管理员查看剩余全部路由
- **AND** 展开后管理员可以继续看到完整的宿主真实公开路径、访问级别与权限标识

#### Scenario: 动态插件未声明路由时不渲染冗余路由块
- **WHEN** 动态插件安装或启用前打开授权审查窗口，但当前 release 未声明任何动态路由合同
- **THEN** 宿主不渲染冗余的路由列表块
- **AND** 现有插件详情与宿主服务授权范围展示仍可正常显示

### Requirement: 动态插件详情弹窗复用路由暴露面审查结构

系统 SHALL 在动态插件详情弹窗中展示当前 release 的动态路由清单，并沿用与授权审查窗口一致的路由信息结构，便于管理员在安装后继续复核插件的对外暴露面。

#### Scenario: 动态插件详情弹窗展示路由信息
- **WHEN** 管理员打开一个声明了动态路由合同的动态插件详情弹窗
- **THEN** 详情弹窗展示独立的“注册路由列表”区块
- **AND** 该区块位于宿主服务信息之后
- **AND** 每条路由至少展示 HTTP 方法、宿主真实公开路径、访问级别、权限标识和摘要等关键信息

#### Scenario: 详情弹窗中的路由信息默认折叠后续条目
- **WHEN** 动态插件详情弹窗中的路由数量大于两条
- **THEN** 详情弹窗默认只展示前两条路由
- **AND** 管理员可以通过“展开”操作查看剩余全部路由

### Requirement: 授权与详情弹窗中的治理分区标题需要清晰可辨

系统 SHALL 在授权弹窗与动态插件详情弹窗中，将“宿主服务授权范围/宿主服务信息”“注册路由列表”等治理分区标题设计为明显区别于正文内容的标题样式，让管理员能够快速识别不同板块边界。

#### Scenario: 授权弹窗中的治理分区标题具备明显视觉层级
- **WHEN** 管理员打开动态插件安装或启用前的授权审查窗口
- **THEN** “宿主服务授权范围”和“注册路由列表”等标题在字号、字重或强调色上明显区别于正文内容
- **AND** 管理员可以快速识别不同治理信息板块的边界

#### Scenario: 详情弹窗中的治理分区标题具备明显视觉层级
- **WHEN** 管理员打开动态插件详情弹窗
- **THEN** “宿主服务信息”和“注册路由列表”等标题在字号、字重或强调色上明显区别于正文内容
- **AND** 标题样式与授权弹窗保持一致的视觉语言

### Requirement: 插件详情弹窗中的长描述不应破坏基础信息布局

系统 SHALL 在插件详情弹窗中优化长描述的展示方式，避免超长插件描述将基础治理信息表格挤压成难以阅读的布局，同时保持描述内容完整可见。

#### Scenario: 长描述在详情弹窗中以独立整行展示
- **WHEN** 插件详情弹窗中的描述内容较长
- **THEN** 宿主将描述信息放在基础信息表格中的独立整行展示，而不是挤压多个半宽信息项
- **AND** 管理员仍然可以完整阅读插件描述内容

### Requirement: 插件详情弹窗隐藏冗余的授权要求字段

系统 SHALL 在插件详情弹窗中隐藏“授权要求”字段，避免与“授权状态”形成冗余展示，并为更有价值的基础信息让出布局空间。

#### Scenario: 详情弹窗中不再单独展示授权要求
- **WHEN** 管理员打开插件详情弹窗
- **THEN** 基础信息表格中不再单独展示“授权要求”字段
- **AND** “授权状态”与“插件描述”等更关键字段仍可正常展示
