## Why

当前动态插件安装/启用前的授权审查窗口只展示宿主服务申请清单，没有同步展示该插件即将对外暴露的动态路由信息。对于管理员来说，`hostServices` 决定插件能访问宿主哪些资源，而动态路由决定插件会新增哪些对外入口；如果缺少后者，管理员就无法在授权前完整评估该插件的外部暴露面，尤其难以及时识别 `public` 路由、登录治理路由和权限绑定范围。

因此，本次迭代需要把动态插件已声明的路由合同纳入授权审查视图，让管理员在安装或启用前即可同时核对“插件会访问什么”和“插件会暴露什么”，提升治理透明度与审查准确性。

## What Changes

- 在动态插件安装/启用前的授权审查窗口中新增“路由信息”展示区，以只读方式展示当前 release 声明的动态路由列表，并将其放在“宿主服务授权范围”之后。
- 授权审查窗口中的路由列表默认只展示前两条，剩余路由通过“展开”操作查看，避免治理弹窗信息过载。
- 路由列表至少展示方法、宿主真实公开路径、访问级别（如 `public` / `login`）、权限标识、摘要等管理员判断暴露面的关键信息。
- 明确展示语义：路由信息属于插件对外暴露面审查，不属于宿主服务授权项；UI 上与 `hostServices` 授权范围并列呈现，但在视觉层级上后置于核心授权信息。
- 扩展宿主插件管理数据投影，使前端授权窗口能够复用后端返回的动态路由清单，而不需要在弹窗打开时额外拼装临时数据。
- 在插件详情弹窗中同步复用相同的动态路由展示结构，便于管理员在安装后继续审查当前 release 的暴露面。
- 优化插件详情弹窗中的“插件描述”展示，让长描述以表格中的独立整行展示，避免基础治理信息表格被挤压变形。
- 强化授权弹窗和详情弹窗中“宿主服务授权范围”“注册路由列表”等板块标题的视觉层级，使其能明确承担分区标题作用。
- 调整插件详情弹窗的基础信息布局，移除“授权要求”字段，并将“插件描述”整合回基础信息表格作为独立整行展示。
- 补充对应规格、前后端实现任务与测试，覆盖无路由、仅登录路由、包含 `public` 路由、含权限标识等典型场景。

## Capabilities

### New Capabilities
- None.

### Modified Capabilities
- `plugin-manifest-lifecycle`: 扩展动态插件安装/启用审查窗口的展示要求，在宿主服务申请清单之外增加动态路由清单展示，并明确路由暴露面与宿主服务授权范围的边界。

## Impact

- **后端接口**: 影响插件管理相关 DTO 与插件列表/详情数据投影，需要新增动态路由清单字段供前端消费。
- **后端服务**: 影响 `apps/lina-core/internal/service/plugin/`、`apps/lina-core/internal/controller/plugin/` 中动态插件 manifest/release 快照的投影逻辑。
- **前端页面**: 影响 `apps/lina-vben/apps/web-antd/src/views/system/plugin/` 下的动态插件安装/启用授权弹窗、插件详情弹窗与相关视图模型。
- **测试验证**: 需要补充前端组件测试或 E2E 用例，并增加后端投影单元测试，确保不同路由合同组合下的展示行为稳定。
