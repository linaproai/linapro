## Why

完整 E2E 回归已经暴露出测试间共享状态竞争：插件生命周期、运行时 i18n 缓存、全局配置、字典、菜单权限等用例在并行执行时可能改变同一数据库或缓存状态，导致协议正确但测试误判。现在需要把这些经验沉淀为可执行的 E2E 治理变更，降低后续全量回归的不稳定性。

## What Changes

- 为 E2E 用例建立全局状态变更分类规则，明确哪些测试必须进入串行边界，哪些测试可以安全并行。
- 扩展测试运行脚本或校验工具，识别插件生命周期、系统参数、字典、菜单权限、运行时 i18n 缓存等高风险共享状态用例。
- 将依赖插件安装、启用、mock 数据和全局前置条件的用例收敛到统一 fixture，避免跨文件隐式依赖。
- 调整缓存/ETag 类测试的断言策略：验证协议语义，而不是假设全量测试期间全局版本不变。
- 建立稳定数据断言规范，业务状态优先使用 API 稳定字段、ID、code 或 labelKey，展示文案单独测试。
- 增加回归材料，记录已识别冲突类型、修复方式、分类规则和后续维护要求。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `e2e-suite-execution-efficiency`: 增强 E2E 串行/并行边界、共享全局状态隔离、缓存重校验断言和 fixture 化前置条件要求。

## Impact

- 影响 `hack/tests/scripts/run-suite.mjs`、`hack/tests/scripts/validate-e2e.mjs`、`hack/tests/fixtures/`、`hack/tests/support/` 和相关 E2E 用例。
- 影响 E2E 运行策略：部分高风险全局状态用例会显式进入串行边界，独立只读或局部数据用例继续保留并行执行。
- 不涉及业务 API 行为变更，不涉及数据库 schema 变更。
- i18n 影响：该变更不新增产品运行时文案；仅涉及测试治理文档和测试断言，若修改测试中语言切换或运行时 i18n 缓存用例，需要明确保持对 `i18n` 行为的语义验证。
