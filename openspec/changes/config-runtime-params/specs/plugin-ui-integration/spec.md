## ADDED Requirements

### Requirement: 插件管理页支持查看插件详情

系统 SHALL 在插件管理页为每条插件记录提供“详情”入口，并通过弹窗展示该插件的完整治理信息。

#### Scenario: 管理员从列表查看插件详情

- **WHEN** 管理员在插件管理页点击某条插件记录的“详情”按钮
- **THEN** 系统打开详情弹窗
- **AND** 弹窗展示该插件的名称、标识、类型、版本、描述、接入状态、当前状态、授权要求、授权状态、安装时间与更新时间

#### Scenario: 动态插件详情页区分宿主服务声明与生效授权

- **WHEN** 当前动态插件存在宿主服务申请清单或授权快照
- **THEN** 详情弹窗按服务类型展示对应的服务类型、方法集合与具体目标条目
- **AND** 当申请清单与授权快照一致时，界面合并为单份当前生效范围展示，而不是重复堆叠两组相同信息
- **AND** 当授权快照与申请清单存在差异时，界面明确区分“申请清单”和“授权快照”
- **AND** 服务信息不再展示“数据表边界”这类弱价值边界标题
- **AND** 当申请清单与授权快照一致时，界面不再额外展示“与申请清单一致”这类重复提示
- **AND** 各资源列表前需要提供一次性的语义说明标题，例如“授权数据表名”“授权访问地址”“授权存储路径”或对应申请态描述
- **AND** 该语义说明标题需要具备与“当前生效范围”同类的背景样式
- **AND** 资源列表中的每个具体条目不再重复追加相同前缀，避免界面冗杂
- **AND** 资源说明标题与其后的资源列表应当与“当前生效范围”及方法标签采用同类的单行流式排列方式，而不是强制换行拆成上下两段
- **AND** 管理员无需离开列表页即可理解当前 release 的声明范围与实际生效授权

#### Scenario: 仅动态插件展示宿主服务空状态提示

- **WHEN** 当前动态插件不存在宿主服务申请清单和授权快照
- **THEN** 详情弹窗显示无宿主服务治理信息的提示
- **AND** 基础信息仍然正常展示
- **AND** 源码插件未声明宿主服务治理信息时不额外展示空状态提示

### Requirement: 动态插件示例页支持分页浏览示例记录

系统 SHALL 在动态插件示例页中为示例记录列表提供分页浏览能力，以便已安装数据量增长后仍可稳定浏览和操作记录。

#### Scenario: 首次进入动态插件示例页时按分页参数加载记录

- **WHEN** 管理员打开 `plugin-demo-dynamic` 的示例页面
- **THEN** 页面调用动态插件示例记录分页接口并携带页码、每页条数参数
- **AND** 页面展示当前页记录、总记录数与当前区间摘要
- **AND** 当总记录数超过单页容量时，页面显示可交互的分页控件

#### Scenario: 管理员切换页码后浏览其他记录

- **WHEN** 管理员在动态插件示例页点击分页控件切换到其他页码
- **THEN** 页面按新的页码重新查询示例记录
- **AND** 表格只展示目标页对应的记录
- **AND** 当前页码与区间摘要同步更新
