## ADDED Requirements

### Requirement: E2E 套件必须提供分层执行入口
E2E 套件 SHALL 同时提供完整回归入口与快速反馈入口,至少覆盖 `smoke`、模块定向运行和 `full` 三类执行方式,使开发者无需每次都手工拼接文件路径或等待整轮全量回归。

#### Scenario: 开发者需要快速高价值反馈
- **WHEN** 开发者执行 smoke 入口
- **THEN** 系统 MUST 运行一组预先维护的高价值关键路径测试文件,且不要求开发者手工指定文件列表

#### Scenario: 开发者只想验证受影响模块
- **WHEN** 开发者执行模块定向入口并提供模块范围
- **THEN** 系统 MUST 根据预定义的模块映射运行对应目录或文件集合,而不是要求人工维护临时 glob

#### Scenario: 完整回归入口仍然可用
- **WHEN** 开发者或 CI 执行完整回归入口
- **THEN** 系统 MUST 运行全部符合约定的 `TC*.ts` 用例文件

### Requirement: 高认证频率用例必须复用预生成登录态
除认证主题本身外,后台管理工作台的高频已登录用例 SHALL 复用预生成的认证态,而不是为每个测试重复执行一次完整 UI 登录流程。

#### Scenario: 普通后台用例默认复用登录态
- **WHEN** 一个测试文件使用管理员已登录页面夹具
- **THEN** 该夹具 MUST 从预生成的 `storageState` 或等价认证态载入页面上下文,而不是在每个测试里重新走登录页

#### Scenario: 认证主题测试仍可验证真实登录链路
- **WHEN** 测试目标是登录、登出、未认证跳转或登录失败等认证行为
- **THEN** 这些测试 MUST 仍然能够显式走真实认证流程,而不是被共享登录态覆盖

#### Scenario: 登录态失效时可以重新生成
- **WHEN** 预生成登录态不存在、过期或失效
- **THEN** 执行准备步骤 MUST 能重新生成可用的认证态供后续测试复用

### Requirement: 高耗时等待必须收敛为状态型等待并建立并行安全边界
E2E 套件 SHALL 优先使用页面、接口和组件状态作为等待信号,并为共享全局状态的测试与可独立执行的测试建立清晰的串行/并行边界。

#### Scenario: 页面对象等待关键元素就绪
- **WHEN** 页面对象等待表格、抽屉、弹窗、路由或提示反馈就绪
- **THEN** 它 MUST 优先使用元素可见性、loading 状态、接口完成或路由变化等确定性信号,而不是固定时长睡眠作为主要策略

#### Scenario: 可独立测试进入并行池
- **WHEN** 一个测试文件不依赖跨文件共享的全局状态
- **THEN** 它 MUST 能被纳入有限 worker 的并行执行池,以缩短整轮回归时间

#### Scenario: 共享全局状态测试保留串行边界
- **WHEN** 一个测试文件会修改插件启停、全局配置、权限矩阵或其他跨文件共享状态
- **THEN** 它 MUST 被明确纳入串行执行边界,避免与其他文件并发运行导致不稳定
