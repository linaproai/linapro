## MODIFIED Requirements

### Requirement: 必须支持数据范围选项

系统 SHALL 支持全部数据、本部门数据和本人创建数据的简化数据范围值。角色数据范围 SHALL 被宿主数据权限治理能力用作当前用户有效数据范围解析的输入，而不只是角色记录上的展示字段。角色数据范围创建、更新、启用、禁用、角色菜单权限变更以及用户角色关系变更后，系统 SHALL 触发访问拓扑或数据权限缓存失效，使后续受保护 API 不会无限期使用旧的数据范围。

#### 场景：配置数据范围

- **当** 角色数据范围设置为支持的值之一时
- **则** 系统存储该值供宿主数据权限治理使用

#### 场景：角色数据范围参与数据权限解析

- **当** 用户拥有启用角色且该角色 `dataScope=2` 时
- **则** 宿主数据权限治理能力将该角色视为本部门数据范围输入
- **且** 接入数据权限治理的宿主资源按本部门范围限制数据

#### 场景：角色数据范围变更触发缓存失效

- **当** 管理员修改角色的 `dataScope` 并保存成功时
- **则** 系统发布访问拓扑或数据权限缓存修订号
- **且** 拥有该角色的用户后续请求不得无限期沿用修改前的数据范围

#### 场景：禁用角色不再贡献数据范围

- **当** 管理员禁用某个角色时
- **则** 与该角色关联的用户登录后不再获得该角色的菜单权限
- **且** 宿主数据权限治理能力不再使用该角色的 `dataScope`
