## ADDED Requirements

### Requirement: 登录后落地路径必须可访问

系统在用户登录成功进入管理工作台时，SHALL 将用户导航到其当前会话下可访问的页面。落地路径 MUST NOT 写死为工作台默认地址（例如 `/dashboard/analytics`）而不校验该路径是否仍在用户可访问菜单与动态路由中。

落地路径解析 MUST 遵循以下优先级：

1. 登录请求携带的合法且当前可访问的 `redirect` 目标
2. 用户信息中的 `homePath`，且该路径在当前会话可访问路由中存在
3. 当前用户左侧可访问菜单中的第一个可导航菜单路径
4. 安全兜底路径（个人中心或租户/平台约定的默认页）

当默认工作台菜单被禁用、隐藏或未授权导致对应路由未注册时，系统 MUST 自动选择上述优先级中的下一个可用路径，而不得停留在 404 页。

#### Scenario: 工作台菜单禁用后登录落到首个可访问菜单

- **WHEN** 管理员已禁用工作台相关菜单（例如分析页/工作台），使用户侧栏不再包含 `/dashboard/analytics`
- **AND** 用户完成登录且未携带可访问的 `redirect` 参数
- **THEN** 系统不得将用户导航到不可访问的 `/dashboard/analytics`
- **AND** 系统将用户导航到左侧可访问菜单中的第一个可导航菜单
- **AND** 用户不会因写死默认首页而看到 404 页面

#### Scenario: 显式 redirect 优先于默认首页

- **WHEN** 用户登录时携带指向当前可访问页面的 `redirect` 参数
- **THEN** 系统登录成功后导航到该 `redirect` 目标
- **AND** 不得强制改写到工作台写死地址

#### Scenario: 显式 redirect 不可访问时降级

- **WHEN** 用户登录时携带的 `redirect` 指向当前会话不可访问或不存在的路径
- **THEN** 系统不得停留在 404
- **AND** 系统按 `homePath` → 首个可访问菜单 → 安全兜底路径 继续解析落地页

#### Scenario: homePath 与动态路由不一致时前端纠偏

- **WHEN** 用户信息返回的 `homePath` 在动态路由装配后不存在于可访问路由集合
- **THEN** 系统在路由装配完成后将落地路径纠正为第一个可访问菜单路径（或安全兜底路径）

## MODIFIED Requirements

### Requirement:登录后返回用户信息

用户登录成功后，系统 SHALL 返回用户信息，包括角色和菜单树。`homePath` MUST 从用户当前可访问的启用菜单中解析，优先选择适合作为落地页的稳定内部路由；当工作台等默认菜单不可用时，`homePath` MUST 指向菜单树中第一个可用的可导航内部路由，而不得返回已停用菜单路径。若用户没有任何可导航内部菜单，系统 MAY 返回安全兜底路径（例如个人中心）。

#### Scenario:登录成功返回用户信息
- **当** 用户使用正确的用户名密码登录时
- **则** 系统返回 userId、username、realName、email、avatar
- **且** 系统返回 roles 字段，包含用户所有角色的 key 列表
- **且** 系统返回 menus 字段，包含用户可访问的菜单树
- **且** 系统返回 permissions 字段，包含用户拥有的权限标识列表
- **且** 系统返回 homePath 字段，指定用户的首页路径，且该路径对应启用且可导航的菜单（或安全兜底路径）

#### Scenario:超级管理员登录
- **当** admin 角色用户登录时
- **则** 系统返回所有菜单（不检查 sys_role_menu 关联）
- **且** roles 包含 "admin"
- **且** permissions 包含 "*:*:*" 表示拥有所有权限

#### Scenario:普通用户登录
- **当** 非超级管理员用户登录时
- **则** 系统根据用户角色查询 sys_role_menu 获取菜单 ID 列表
- **且** 系统根据菜单 ID 列表构建菜单树
- **且** 系统过滤掉状态为停用（status = 0）的菜单
- **且** 系统过滤掉隐藏的菜单（visible = 0）

#### Scenario:用户登录时无角色
- **当** 用户登录时未分配任何角色
- **则** 系统返回空菜单树
- **且** roles 为空数组
- **且** permissions 为空数组

#### Scenario:用户所有角色被停用
- **当** 用户的所有角色都被停用时
- **则** 系统返回空菜单树
- **且** roles 为空数组
- **且** permissions 为空数组

#### Scenario:工作台菜单停用时 homePath 避开工作台
- **WHEN** 用户可访问菜单中不包含工作台叶子菜单（例如分析页路径已停用）
- **AND** 仍存在其他可导航内部菜单
- **THEN** 系统返回的 `homePath` 不得为已停用的工作台路径
- **AND** `homePath` 为菜单顺序中第一个可导航内部路由
