## Context

本次变更来源于人工回归测试，反馈横跨宿主后端、源码插件、默认管理工作台、运行时 i18n 资源和 E2E 测试。当前项目处于全新项目阶段，默认交付数据可以直接在现有初始化 SQL 和 packed manifest 资源中整理，后续通过重新初始化数据库获得一致结果。

现有 i18n 基础能力已经支持按请求语言投影菜单、字典、配置、角色等后端拥有的数据，但回归反馈表明部分页面仍存在三类缺口：资源键缺失或投影范围不足、英文长文案导致布局不可读、前端交互未覆盖关键操作确认。

## Goals / Non-Goals

**Goals:**

- 为 12 条人工反馈建立独立可追踪任务，并完成检查、修复和验证。
- 修正监控采集默认间隔，将默认上报周期统一为 1 分钟。
- 清理默认交付页面和种子数据在英文环境下的中文系统文案残留。
- 保持后端拥有的数据由后端或插件资源本地化，前端只消费接口返回值和运行时语言包。
- 调整英文环境下表单、表格和树形列表的可读性与交互。
- 用 Playwright 截图/断言覆盖用户反馈中特别指出的英文页面和服务监控磁盘表格。

**Non-Goals:**

- 不引入新的运行时 i18n 存储模型，不增加新的数据库表。
- 不把用户创建或可自由编辑的业务数据自动翻译为英文。
- 不重构 Vben、VXE Grid 或 Ant Design Vue 的通用组件。
- 不改变动态插件权限模型，只修正动态路由按钮菜单的挂载和展示语义。

## Decisions

1. **默认交付数据直接整理现有 seed 和 packed 资源。**  
   因项目当前无历史兼容负担，初始化 SQL、默认配置模板和 packed manifest 副本直接改为新的交付基线。这样可以保证 `make init` 后的默认数据与源码资源一致，避免用新迁移文件覆盖同一批初始种子数据。

2. **后端拥有的数据继续由所属模块投影。**  
   角色、配置、菜单、字典、部门虚拟节点等后端拥有的展示字段由对应宿主或插件服务按当前语言输出；前端不新增按中文值映射英文的临时表，避免破坏 i18n 单一事实源。

3. **英文布局问题优先在页面局部调整。**  
   岗位、字典、服务监控等页面只调整相关表单 schema、label 宽度、列宽或表格容器约束，不做全局表单和表格系统改造，降低影响面。

4. **动态插件按钮权限归属按菜单锚点修正。**  
   动态路由权限按钮应挂在所属动态插件页面菜单下；若动态路由没有明确页面锚点，则挂到该插件主菜单下，而不是生成与业务菜单脱节的“Dynamic Route Permission:*”漂浮按钮。

5. **立即执行确认复用现有危险操作确认能力但区分语义样式。**  
   前端使用与删除一致的 Popconfirm/确认组件链路，文案和按钮样式体现“执行动作”而非“删除动作”，并保持权限按钮逻辑不变。

## Risks / Trade-offs

- **默认 SQL 与 packed 副本不一致** → 修改后同时检查 `manifest/` 与 `internal/packed/manifest/`，并运行资源搜索验证残留中文和旧默认值。
- **英文种子数据与可编辑主数据边界混淆** → 仅对框架默认交付、受保护或系统生成内容做本地化投影；普通用户后续创建的数据保持数据库原值。
- **表格列宽调整影响窄屏布局** → 优先在桌面英文回归场景设置最小宽度和 tooltip/nowrap，不移除响应式横向滚动能力。
- **动态插件权限挂载影响角色授权树** → 补充菜单树/角色树相关单元测试或 E2E 验证，确保按钮仍参与角色授权。
- **E2E 运行成本较高** → 新增用例按模块拆分，优先运行本变更相关用例，再按需要运行完整套件。
