## Why

`apps/lina-core/internal/service/plugin`根包已经成为插件宿主能力的公共 facade，但同时承载了大量生命周期细节、运行时升级编排、列表投影、host service 配置和单元测试 fixture。根包 Go 文件数量和测试文件体量持续增长，会增加后续插件能力扩展、审查和回归验证成本。

本次变更按“面向可持续交付的 AI 原生全栈框架”定位，对插件服务代码组织做内部治理：保留根包稳定 facade 契约，把可机械收敛的小文件、重复测试 helper 和主题混杂测试文件整理到更清晰的位置。

## What Changes

- 合并同一职责下的低行数 Go 文件，减少只有 alias、配置入口或状态转发的小文件。
- 将根插件服务测试中复用的列表查找、字符串包含等 helper 收敛到明确的测试支撑文件。
- 将大测试文件按被测源码职责拆分，降低单文件维护成本，并保持测试自包含、顺序无关。
- 保持根插件`Service`公开契约、HTTP API、数据库模型、插件 manifest、动态插件 ABI 和用户可见行为不变。

## Impact

- 影响`apps/lina-core/internal/service/plugin`及其`internal/*`子组件的 Go 源码和单元测试文件组织。
- 不新增 HTTP API、路由、DTO、SQL、前端页面、用户可见文案或语言包资源。
- `i18n`影响：无运行时文案、API 文档源文本、插件清单或语言包资源变更。
- 缓存一致性影响：仅移动和整理既有缓存相关代码，不改变权威数据源、失效触发点、跨实例同步机制或最大陈旧时间。
- 数据权限影响：不新增列表、详情、导出、下载、写操作或插件数据访问路径；既有授权和数据权限语义不变。
- 开发工具跨平台影响：不新增脚本、生成入口、`linactl`命令或 CI 配置。
