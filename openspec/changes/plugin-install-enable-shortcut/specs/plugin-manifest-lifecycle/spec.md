## ADDED Requirements

### Requirement: 插件安装审查链路支持直接串联启用

系统 SHALL 允许管理员在插件安装审查链路中直接触发后续启用动作，但宿主仍必须按既有 `install -> enable` 生命周期顺序执行，而不是把两者折叠成新的隐式状态迁移。

#### Scenario: 在安装审查弹窗中选择安装并启用
- **WHEN** 管理员在未安装插件的安装审查弹窗中选择“安装并启用”
- **THEN** 宿主先执行该插件的安装生命周期
- **AND** 安装成功后再继续执行启用生命周期
- **AND** 两步都成功时，插件最终状态为“已安装且已启用”

#### Scenario: 动态插件组合动作复用安装阶段授权快照
- **WHEN** 动态插件在安装审查弹窗中完成宿主服务授权确认，并继续选择“安装并启用”
- **THEN** 宿主在安装阶段持久化当前 release 的授权快照
- **AND** 随后的启用步骤直接复用该快照
- **AND** 组合动作过程中 MUST NOT 再次弹出第二个授权确认窗口

#### Scenario: 组合动作的启用失败不回滚已成功的安装
- **WHEN** 管理员执行“安装并启用”，且安装生命周期已经成功完成，但启用生命周期失败
- **THEN** 宿主保留该插件“已安装、未启用”的真实状态
- **AND** 宿主 MUST NOT 因为启用失败而自动撤销已经完成的安装结果
- **AND** 管理员后续仍可基于现有安装结果再次触发启用
