## Why

参数设置页当前将所有参数值统一为自由文本（`Textarea`）编辑，枚举类参数（如 `true/false`、布局枚举、主题模式）只能依赖备注说明靠人工拷贝粘贴，易误填且交互差。参数项需要具备明确的输入类型与可选值元数据，使新增/编辑时按类型渲染对应前端组件，降低运维错误并提升可配置体验。

## What Changes

- 为 `sys_config` 增加参数值类型（`value_type`）与类型元数据（`options`，用于下拉/单选/多选的选项列表等）。
- 扩展参数 CRUD / 详情 / 列表 API，使管理面可读写类型与选项；创建与更新时按类型校验参数值。
- 参数设置新增/编辑表单按 `value_type` 动态切换输入组件（文本、多行文本、数字、布尔、下拉、单选、多选、富文本等）。
- 为内置种子参数补齐合理类型与选项（如开关、布局枚举、主题模式、数字上限等）。
- 导入/导出与导入模板同步支持类型与选项列；历史无类型数据默认按 `text` 兼容。
- 运行时配置读取路径仍以字符串 `value` 为权威业务值，不因类型元数据改变宿主热路径语义。

## Capabilities

### New Capabilities

- `config-param-value-types`：参数值类型模型、选项元数据、类型化校验与管理面按类型渲染编辑组件。

### Modified Capabilities

- `config-management`：参数列表/详情/创建/更新/导入导出契约扩展类型与选项字段；内置参数种子携带类型元数据；导入导出列头翻译扩展。

## Impact

- **数据库**：`sys_config` 在 `005-config-management.sql` 建表即含 `value_type`、`options`；内置参数类型元数据写入 `005`/`011` 的 Seed，不保留独立增量 SQL。
- **后端**：`api/config`、`sysconfig` 服务、实体/DAO（`make dao`）、导入导出与值校验逻辑。
- **前端**：`views/system/config` 表单 schema 改为按类型动态组件；API 模型扩展；i18n 文案与类型选项标签。
- **E2E / 单测**：覆盖类型化编辑、枚举选择、非法值拒绝；导入导出列扩展。
- **插件 / 宿主读路径**：`HostConfig.GetRaw` / 运行时快照仍消费字符串 `value`；插件 `SetValue` 不强制要求类型（缺省 `text`）。
- **非目标**：不把参数类型做成独立业务模块；不改造插件专属设置页表单体系；不引入字典类型外链作为本迭代必选项。
