## ADDED Requirements

### Requirement: lina-core 核心宿主能力保持通用边界
系统 SHALL 将 `apps/lina-core` 视为框架核心宿主服务，并优先保障通用模块接口能力、组件能力、系统治理能力和插件扩展能力的稳定性与可复用性。

#### Scenario: 页面需求仅影响工作台展示
- **WHEN** 一个需求只改变表格列、筛选项、树选择器、工作台聚合、路由装配或其他具体页面展示结构
- **THEN** 系统优先通过工作台适配接口或前端适配层完成该变更
- **AND** 不因该页面需求直接修改 `lina-core` 的核心领域契约、通用 service 语义或存储模型

#### Scenario: 计划修改核心接口或模型
- **WHEN** 开发者计划因某个前端页面需求修改 `lina-core` 的核心接口、领域模型或持久化结构
- **THEN** 该修改必须能够证明其服务于框架级通用能力而非单一页面形态
- **AND** 若无法证明，则应回退为工作台适配实现方案

### Requirement: 工作台适配接口必须被显式分类
系统 SHALL 将菜单路由投影、当前用户工作台启动数据、树选择器、下拉选项等明显面向工作台装配的输出显式归类为工作台适配接口，而不是通用领域接口。

#### Scenario: 接口返回工作台装配数据
- **WHEN** 一个接口返回菜单路由、宿主工作区启动数据、树选择器节点或下拉选项等工作台装配数据
- **THEN** 其接口说明、DTO 注释和相关规范明确标注该输出属于工作台适配语义
- **AND** 不将该输出描述为通用领域模型本身

#### Scenario: 通用领域能力被多个工作台复用
- **WHEN** 一个能力同时需要被不同工作台或不同接入方式消费
- **THEN** 系统优先保留稳定的通用领域接口
- **AND** 不同工作台所需的菜单、路由、展示结构或聚合视图通过独立适配输出组装
