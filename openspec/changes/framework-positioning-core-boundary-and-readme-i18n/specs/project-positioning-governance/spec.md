## ADDED Requirements

### Requirement: 项目定位统一
系统 SHALL 将 `Lina` 的统一项目定位定义为“`AI驱动的全栈开发框架`”，并在项目规范、系统元数据、入口文案和仓库说明中保持该定位一致。

#### Scenario: 项目级说明使用统一定位
- **WHEN** 开发者编写或更新项目规范、系统元数据、入口文案或仓库说明
- **THEN** 相关内容使用“`AI驱动的全栈开发框架`”描述 `Lina`
- **AND** 不再将 `Lina` 定义为“仅一个后台管理系统”或等价表述

#### Scenario: 管理后台能力作为从属能力表述
- **WHEN** 文档、页面说明或系统元数据需要提及菜单、角色、工作区、系统管理等后台能力
- **THEN** 这些能力被表述为 `Lina` 的默认管理工作台或内建通用模块
- **AND** 不被表述为 `Lina` 的唯一产品边界

### Requirement: 项目级入口文案保持一致
系统 SHALL 要求仓库入口文档、系统信息页、接口文档元数据和脚本输出中的项目介绍保持一致的定位语义。

#### Scenario: 多个入口同时展示项目介绍
- **WHEN** 用户查看仓库根说明文档、系统信息页、系统接口文档或项目脚本输出
- **THEN** 不同入口中的项目介绍保持一致的定位语义
- **AND** 不出现一个入口描述为全栈开发框架而另一个入口仍描述为后台管理系统的冲突情况
