## MODIFIED Requirements

### Requirement: 插件生命周期状态机可治理

系统 SHALL 为插件提供可审计的生命周期状态机，并按源码插件与动态插件区分生命周期语义。

#### Scenario: 源码插件随宿主编译集成

- **WHEN** 宿主编译源码插件所在的源码树并生成 LinaPro 二进制
- **THEN** 源码插件的后端 Go 代码与宿主源码一起完成编译
- **AND** 源码插件在插件注册表中保持“可发现但默认未安装”的治理形态，而不是自动视为已接入
- **AND** 管理员需要先显式安装源码插件，再决定是否启用

#### Scenario: 源码插件首次同步后默认未安装

- **WHEN** 宿主首次发现一个源码插件并将其写入插件注册表
- **THEN** 该源码插件默认处于“未安装且未启用”状态
- **AND** 宿主同步只更新 manifest、release 快照与基础治理元数据，不会自动执行安装 SQL、初始化逻辑或菜单注册
- **AND** 宿主后续同步不会覆盖管理员对该源码插件做出的显式安装、卸载、启用或禁用结果

#### Scenario: 安装源码插件

- **WHEN** 管理员安装一个已发现的源码插件
- **THEN** 宿主按清单顺序执行该源码插件声明的安装 SQL 与初始化步骤
- **AND** 宿主将插件注册表和当前版本记录写入“已安装”状态，并同步该插件声明的菜单、资源引用与治理关系
- **AND** 插件在显式启用前不会对普通用户可见

#### Scenario: 安装动态插件

- **WHEN** 管理员安装一个合法的 `wasm` 动态插件
- **THEN** 宿主创建插件安装记录与当前版本记录
- **AND** 宿主按清单依次处理迁移、资源注册、权限接入与前后端装载准备
- **AND** 插件在显式启用前不会对普通用户可见

#### Scenario: 禁用插件

- **WHEN** 管理员将已启用插件切换为禁用状态
- **THEN** 宿主停止该插件的 Hook、Slot、页面与菜单暴露
- **AND** 宿主保留插件业务数据、角色授权关系与安装记录
- **AND** 插件重新启用后可以恢复既有治理关系

#### Scenario: 卸载源码插件

- **WHEN** 管理员卸载一个已安装的源码插件
- **THEN** 宿主先停用该插件，并向管理员确认是否在卸载时同时清理该插件的自有存储数据
- **AND** 管理员选择清理存储数据时，宿主再执行该源码插件声明的卸载 SQL 与卸载阶段初始化逻辑，用于清理插件自有业务表数据、业务表结构或关联存储文件
- **AND** 管理员未选择清理存储数据时，宿主跳过该源码插件的卸载 SQL，但仍继续完成治理卸载
- **AND** 宿主移除该插件在宿主侧注册的菜单、资源引用与治理挂载信息
- **AND** 宿主保留源码插件的编译产物与清单可发现性，使其后续仍可再次安装
- **AND** 源码插件仅在“禁用”阶段保留既有业务数据；一旦管理员在卸载确认中选择清理存储数据，插件数据与相关文件必须被同步清除

#### Scenario: 卸载动态插件

- **WHEN** 管理员卸载一个动态插件
- **THEN** 宿主先停用该插件，并向管理员确认是否在卸载时同时清理该动态插件的自有存储数据
- **AND** 管理员选择清理存储数据时，宿主执行该动态插件声明的卸载 SQL，并按宿主确认授权的 storage paths 清理插件自有存储文件
- **AND** 管理员未选择清理存储数据时，宿主跳过该动态插件的卸载 SQL 与存储清理，但仍继续完成治理卸载
- **AND** 宿主移除该插件在宿主侧注册的菜单、资源引用、运行时产物与挂载信息
- **AND** 动态插件仅在“禁用”阶段保留既有业务数据；一旦管理员在卸载确认中选择清理存储数据，插件业务表数据与相关文件必须被同步清除
- **AND** 宿主保留卸载审计信息

### Requirement: 插件菜单通过清单元数据治理

系统 SHALL 使用 `plugin.yaml` 或动态产物嵌入 manifest 中的 `menus` 元数据管理插件菜单和按钮权限，而不是要求插件通过 SQL 直接操作 `sys_menu` 与 `sys_role_menu`。

#### Scenario: 源码插件安装时同步菜单

- **WHEN** 管理员安装一个源码插件
- **THEN** 宿主在执行该插件安装 SQL 后，依据其 `menus` 元数据幂等写入或更新对应的 `sys_menu`
- **AND** 宿主依据 `parent_key` 解析真实 `parent_id`
- **AND** 宿主为这些菜单补齐默认管理员角色授权，而不要求插件 SQL 再手工写入 `sys_role_menu`

#### Scenario: 安装动态插件注册菜单

- **WHEN** 管理员安装一个动态插件
- **THEN** 宿主在执行插件安装 SQL 后，继续依据 manifest `menus` 元数据幂等写入或更新对应的 `sys_menu`
- **AND** 插件安装 SQL 可以继续负责业务表与业务种子数据，但不再承担插件菜单注册职责

#### Scenario: 源码插件卸载时删除菜单

- **WHEN** 管理员卸载一个源码插件
- **THEN** 宿主在该插件卸载 SQL 成功执行后，依据 manifest `menus` 元数据删除对应的 `sys_role_menu` 关联与 `sys_menu`
- **AND** 删除范围仅限该源码插件 manifest 中声明的菜单键，不依赖插件 SQL 手工维护删除语句
- **AND** 若插件未声明任何菜单，宿主跳过菜单删除步骤

#### Scenario: 卸载动态插件删除菜单

- **WHEN** 管理员卸载一个动态插件
- **THEN** 宿主在动态插件治理卸载过程中，依据 manifest `menus` 元数据删除对应的 `sys_role_menu` 关联与 `sys_menu`
- **AND** 删除范围仅限该插件 manifest 中声明的菜单键，不依赖插件 SQL 手工维护删除语句
- **AND** 若插件未声明任何菜单，宿主跳过菜单删除步骤

#### Scenario: 升级插件

- **WHEN** 管理员为已安装插件安装更高版本的 release
- **THEN** 宿主为插件创建新的 release 记录与代际信息
- **AND** 旧 release 在新 release 生效前保持可回退
- **AND** 升级失败时宿主能够回滚到上一个可用 release

#### Scenario: 升级失败 release 保持隔离

- **WHEN** 一个动态插件在升级、迁移或前端资源切换过程中失败并触发回滚
- **THEN** 宿主将失败 release 标记为 `failed`
- **AND** 宿主恢复注册表指针到上一个稳定 release
- **AND** 失败 release 的公共前端资源不会继续通过 `/plugin-assets/<plugin-id>/<version>/...` 对外提供

#### Scenario: 源码插件显式暴露安装卸载操作

- **WHEN** 管理员查看源码插件的插件管理操作项
- **THEN** 宿主在源码插件未安装时展示“安装”操作，在源码插件已安装时展示“卸载”操作
- **AND** 源码插件的启用/禁用操作仅在已安装后可用
