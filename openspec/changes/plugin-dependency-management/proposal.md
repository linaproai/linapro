## Why

当前 `plugin.yaml` 只能描述插件自身元数据、运行时资源和治理资源，无法声明插件依赖关系，也无法表达插件对 LinaPro 框架版本的要求。随着官方源码插件和动态插件数量增加，插件安装顺序、依赖缺失、版本不满足和卸载破坏下游插件的问题需要在清单和生命周期层显式治理。

## What Changes

- 在插件清单中新增 `dependencies` 声明，支持 LinaPro 框架版本约束和插件依赖约束。
- 安装插件前解析依赖图，校验框架版本、依赖插件存在性、依赖插件版本范围、自依赖和循环依赖。
- 支持对已发现且版本满足的硬依赖按拓扑顺序自动安装，安装结果必须可追踪且失败时返回依赖链上下文。
- 卸载插件前检查反向依赖，防止卸载仍被其他已安装插件硬依赖的插件。
- 启动自动启用流程必须复用依赖解析和安装编排，确保 `plugin.autoEnable` 中的插件及其自动依赖按确定性顺序收敛。
- 源码插件升级与动态插件升级必须校验新版本依赖约束，避免升级后破坏已安装插件的依赖合同。
- 插件管理 UI 和 API 需要展示依赖检查结果、自动安装计划和阻断原因。

## Capabilities

### New Capabilities

- `plugin-dependency-management`: 定义插件清单依赖声明、框架版本约束、依赖解析、自动安装、反向依赖保护和依赖检查结果展示。

### Modified Capabilities

- `plugin-manifest-lifecycle`: 插件清单校验和显式插件安装生命周期需要识别并执行依赖声明。
- `plugin-startup-bootstrap`: 启动自动启用流程需要在安装和启用目标插件前解析并安装自动依赖。
- `plugin-upgrade-governance`: 插件升级需要校验新发布的依赖约束，避免产生不满足依赖的有效版本。

## Impact

- 影响后端插件清单类型、清单校验、插件发现、插件安装/卸载/启用、源码插件升级、动态插件发布安装和启动自动启用流程。
- 影响插件管理 API：安装前依赖检查、安装响应、阻断错误和插件列表/详情中的依赖状态展示。
- 影响前端插件管理页面：安装确认、依赖计划展示、阻断原因、反向依赖卸载提示，需要同步维护前端运行时 i18n 和宿主 manifest i18n。
- 影响动态插件产物 manifest 自定义段和源码插件 `plugin.yaml` 样例，需要保持 source/dynamic 两种插件清单语义一致。
- 不新增业务数据权限边界；依赖检查读取插件 registry 和发布快照，不暴露租户业务数据。
- 涉及插件 registry、release snapshot、enabled snapshot、runtime bundle 和 i18n bundle 的缓存一致性；依赖安装带来的多插件状态变化必须复用现有插件 runtime revision/event 失效机制。
