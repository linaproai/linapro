## Why

默认管理工作台的前端体验需要同时收敛两类高频问题：列表搜索区无实际收益的展开/折叠控件噪音，以及主题切换时把整页快照动画、主题变量全量重算、图表重建和表格语言刷新压到同一次主线程更新造成的卡顿。

搜索区方面，无论是否存在可折叠内容都会显示右侧展开/折叠按钮，一行搜索条件页面上该按钮没有收益。主题方面，项目已具备无需重建图表的 `ECharts 6` 动态主题能力，应收敛更新链路并建立可回归的性能约束，同时保留原有圆形揭幕视觉语义。

## What Changes

- 调整默认管理工作台中基于 `useVbenVxeGrid` 和 `VbenForm` 的搜索表单行为，按实际渲染行数决定是否显示展开/折叠按钮；多行场景保留折叠能力，不改变搜索、重置、表格刷新和搜索面板显示/隐藏语义。
- 行数判断响应断点、动态 `schema`、语言标签宽度、搜索区显示状态和表单可见性变化。
- 保留主题圆形揭幕双向方向与节奏；主题事务期间暂停页面内容自身颜色过渡，避免快照内分帧换色。
- 保持偏好分支引用稳定，并将主题副作用限定到补丁实际包含的 CSS 变量；用 `ECharts 6` 动态主题复用活动图表实例，维护完整业务配置基线，禁止清空/销毁/重建；拆分 `VXE` 主题与语言监听。
- 在原生快照回调前同步保存用户点击的目标主题，使点击后立即刷新也使用目标主题；重叠点击按最后一次意图提交。
- 增加共享逻辑单元测试、搜索折叠 E2E 与后台分析页主题切换性能 E2E 回归。

## Capabilities

### New Capabilities

- `frontend-theme-switching`：定义后台管理端主题切换的实例稳定性、更新范围、持久化语义和可观察性能要求。

### Modified Capabilities

- `base-layout`：默认管理工作台的列表搜索区应仅在存在可折叠的多行搜索条件时显示展开/折叠按钮。

## Impact

- 影响代码：`form-ui` 表单渲染与操作区、`vxe-table` 表格搜索表单集成、主题切换按钮、偏好管理器、主题变量更新器、设计系统适配器，以及 `ECharts`/`VXE` 共享插件与宿主 E2E。
- 前端行为：一行搜索栏隐藏展开/折叠按钮；多行搜索栏继续显示并可折叠；主题切换保持揭幕视觉、实例稳定与可回归性能门禁。
- `i18n` 影响：不新增运行时文案或翻译键；复用现有 `expand`/`collapse` 文案，仅调整显示条件；主题切换不修改用户可见文案。
- 测试影响：搜索折叠 E2E TC006；主题切换单元测试与分析页 E2E TC007。
- 无 API、数据库、后端 Go、数据权限、缓存一致性、开发工具跨平台或插件资源影响。
