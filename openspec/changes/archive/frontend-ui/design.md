# Design

## 列表搜索区展开折叠按钮

默认管理工作台的列表页普遍通过 `useVbenVxeGrid` 接入搜索表单。原先表格封装默认 `showCollapseButton: true`，一行搜索条件也会出现无收益的展开/折叠入口。

实现在公共 `form-ui` 行数计算链路中派生“是否存在可折叠内容”，再交给操作区决定是否渲染按钮。真实行数依赖 DOM 布局、断点、`schema` 与可见性，不能仅按字段数量推断。按钮可见性与折叠行为一致：仅当当前布局下存在超出 `collapsedRows` 的字段时显示；折叠不会隐藏任何字段时即使配置为 `true` 也不渲染。

不新增按钮文案；多行场景继续使用现有 `expand`/`collapse`。不逐页改搜索配置，不新增后端接口、权限、缓存或插件运行时能力。表单初次不可见或搜索面板隐藏时需在可见后重新计算行数。

## 主题圆形揭幕与事务

支持 `View Transition API` 且用户未要求减少动态效果时，主题按钮从按钮中心启动圆形揭幕，鼠标与键盘触发使用相同圆心。亮转暗时 `::view-transition-old(root)` 置于上层并从覆盖视口的圆收缩到按钮位置，暗色从左下向右上显现；暗转亮时 `::view-transition-new(root)` 置于上层并从按钮位置扩展，亮色从右上向左下展开。节奏保持 `450ms ease-in`。动画结束后由原生生命周期清理快照，不再 `skipTransition()` 强制跳过尾帧。

模式更新开始时根节点进入持续一个绘制周期的主题事务，暂停按钮以外页面内容的 `transition`，保证新快照中组件颜色同一帧落到目标主题。减少动态效果偏好下直接切换并关闭局部动画。每次点击分配递增序号，过期快照/`ready` 回调不再提交状态或启动动画，重叠点击最终落到最后一次用户意图。

## ECharts 与 VXE 主题消费者

活动图表首次完整渲染通过 `setOption(full, { notMerge: true })` 建立动态主题恢复基线，并在应用侧保存完整原始业务配置 `canonicalOptions`。`updateData()` 只接受“已有系列、稳定唯一显式 `id`、仅含 `id` 与 `data`”的增量并折叠进基线；结构变化必须完整配置 + `notMerge: true`。主题切换时以 `notMerge: true`、`lazyUpdate: true` 提交基线后立即 `setTheme()`，保持同一实例，不调用 `clear()`、`dispose()`、`init()` 或 `resize()`。`KeepAlive` 停用的图表不进入切换关键路径，重新激活时只应用一次当前主题。

`VXE` 使用两个独立监听器：主题监听器只 `setTheme()`，语言监听器只加载和设置语言。`Ant Design Vue` 继续根据实际主题与设计变量更新。

## 主题补丁范围与持久化

保留 `updatePreferences()` 分支级 `defu` 合并，用测试锁定未变化分支的引用稳定性。主题更新器同时接收完整当前主题和本次补丁，以补丁字段决定 DOM 更新范围：模式变化只切换根节点 `.dark`；仅当主色或颜色字段变化时生成对应色阶；圆角与字体大小分别按自身字段更新。

主题按钮在 `startViewTransition()` 前同步写入目标主题提示值和用户显式选择标记；运行时主题仍只在快照更新回调中提交。完整偏好快照继续 `150ms` 防抖保存，且防抖保存不回写主题/语言提示，避免旧队列覆盖最新点击意图。加载完整快照时专用主题与语言提示值优先级更高。一致性模型为单浏览器最终一致：提示值立即一致，完整快照最大陈旧 `150ms`。

## Risks / Trade-offs

- 搜索区行数在表单不可见时可能不准 → 可见后重新计算；E2E 覆盖一行/多行与布局无重叠。
- 业务图表显式颜色可能覆盖动态主题默认值 → 保持业务配置语义，并用三态画布像素与截图验证可读性。
- 专用主题提示与完整快照短暂不同 → 加载时始终以提示覆盖快照，并测试立即刷新。
- 原生文档快照仍有固定合成成本 → 收窄回调内变量/图表/表格/持久化工作，E2E 记录耗时门禁。
- 连续快速点击产生重叠原生快照 → 递增序号使过期回调失效。

## Migration Plan

纯前端行为修复：先替换共享主题状态与组件更新路径并跑单元测试，再恢复双向揭幕并新增分析页 E2E；搜索折叠随公共表单层一并生效。回滚按文件恢复即可，无需数据迁移或缓存清理。
