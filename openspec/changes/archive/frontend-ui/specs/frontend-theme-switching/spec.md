# frontend-theme-switching

## Purpose

定义后台管理端主题切换的视觉语义、组件实例稳定性、更新范围、持久化与可观察性能约束。

## Requirements

### Requirement: 主题圆形揭幕必须保持双向方向

后台管理端在浏览器支持`View Transition API`且用户未要求减少动态效果时，`MUST`从主题按钮位置执行圆形揭幕。亮转暗`MUST`让旧亮色快照向右上按钮收拢，使暗色从左下向右上显现；暗转亮`MUST`让新亮色快照从右上按钮向左下展开。快照中的页面内容`MUST`在同一绘制帧完成颜色切换。

#### Scenario: 用户从亮色切换到暗色

- **WHEN**用户点击顶部主题按钮从亮色切换到暗色
- **THEN**系统将根节点切换为暗色主题，并在`::view-transition-old(root)`上把旧亮色快照从覆盖视口的圆收缩到按钮位置
- **AND**暗色内容在视觉上从左下向右上显现
- **AND**页面内容暂停自身颜色过渡并在同一绘制帧完成换色
- **AND**页面保持可交互，主题按钮仅执行局部图标反馈

#### Scenario: 用户从暗色切换回亮色

- **WHEN**用户点击顶部主题按钮从暗色切换回亮色
- **THEN**系统在`::view-transition-new(root)`上把新亮色快照从按钮位置扩展到覆盖视口
- **AND**亮色内容在视觉上从右上向左下展开
- **AND**页面内容在同一绘制帧完成换色且不出现中间色或空白帧

#### Scenario: 用户偏好减少动态效果

- **WHEN**浏览器声明`prefers-reduced-motion: reduce`
- **THEN**系统不执行文档圆形揭幕或按钮局部过渡动画
- **AND**主题状态仍直接、完整地切换

### Requirement: 主题相关组件必须保持实例稳定

主题模式变化时，系统`MUST`复用已活动的图表实例，并且`MUST NOT`触发与主题无关的表格语言刷新。

#### Scenario: 分析页包含多个活动图表

- **WHEN**用户在分析页切换主题
- **THEN**每个活动`ECharts`实例通过动态主题接口更新
- **AND**图表实例标识保持不变
- **AND**系统以完整原始业务配置建立新的恢复基线，并与动态主题在一次统一绘制中生效，画布保持非空
- **AND**系统不销毁、重新初始化或调整图表尺寸

#### Scenario: 图表持续接收数据增量

- **WHEN**调用方使用稳定且唯一的系列`id`更新已有系列的`data`
- **THEN**系统把最新数据折叠进完整业务配置基线且不保留历史补丁日志
- **AND**主题切换耗时与历史数据更新次数无关
- **AND**包含结构变化或含糊系列匹配的更新必须使用完整配置和`notMerge: true`

#### Scenario: 图表所在标签页已被停用

- **WHEN**用户切换主题时某个已创建的`ECharts`实例处于`KeepAlive`停用状态
- **THEN**该实例不在主题切换关键路径中执行换肤或重绘
- **AND**标签页重新激活时，该实例只应用一次当前主题并保留原业务数据

#### Scenario: 表格页切换主题

- **WHEN**用户在已初始化`VXE`的页面切换主题且语言未变化
- **THEN**系统只更新`VXE`主题
- **AND**系统不加载语言资源、不重置国际化消息且不重新设置语言

### Requirement: 主题更新必须限制到变化字段

偏好管理器`MUST`只合并本次更新涉及的顶层分支，并根据主题补丁字段更新对应`DOM`状态。未变化的偏好分支、颜色变量、圆角和字体大小`MUST`保持不变。

#### Scenario: 仅更新默认主题模式

- **WHEN**偏好更新只包含`theme.mode`且当前内置主题的亮暗主色相同
- **THEN**系统只切换根节点主题类和必要的主题消费者
- **AND**系统不重新生成颜色色阶、不重写圆角且不重写字体大小
- **AND**所有未变化的顶层偏好分支保持原对象引用

#### Scenario: 亮暗模式使用不同主色

- **WHEN**偏好更新包含`theme.mode`且当前内置主题为亮暗模式定义了不同主色
- **THEN**系统只重新生成并应用主色色阶
- **AND**成功色、警告色、错误色、圆角和字体大小保持不变

### Requirement: 显式主题偏好必须支持立即刷新恢复

用户显式主题模式`MUST`在点击关键路径中保存首屏所需的主题提示值，完整偏好快照`MAY`防抖保存。加载时专用主题提示值`MUST`覆盖尚未刷新的完整快照，并继续覆盖公共前端默认主题。

#### Scenario: 切换主题后立即刷新

- **WHEN**用户显式切换主题并在完整偏好防抖保存完成前刷新页面
- **THEN**首屏和应用初始化均使用最新显式主题
- **AND**页面不回退到旧快照或公共前端默认主题
- **AND**更早排队的完整偏好防抖保存不会覆盖最新主题提示值

#### Scenario: 用户连续快速切换主题

- **WHEN**前一次原生主题快照尚未提交时用户再次点击主题按钮
- **THEN**过期的快照回调不提交主题状态或启动圆形动画
- **AND**最终运行时主题与持久化提示值均等于最后一次点击意图

#### Scenario: 用户没有显式主题偏好

- **WHEN**本地不存在用户主题覆盖标记
- **THEN**系统继续使用公共前端配置或浏览器默认主题

### Requirement: 主题揭幕关键路径必须受性能门禁约束

后台分析页的主题切换回归环境中，系统`MUST`限制原生快照更新回调、动画就绪和动画完成耗时，并在动画开始边界保持活动图表画布非空。

#### Scenario: 分析页执行双向主题揭幕

- **WHEN**分析页从亮色切换到暗色或从暗色切换到亮色
- **THEN**原生快照更新回调在`200ms`内完成
- **AND**点击到动画就绪在`300ms`内完成
- **AND**包括`450ms`揭幕动画在内的总耗时在`1000ms`内完成
- **AND**动画就绪时全部活动图表实例标识保持不变且每个画布已有非单色绘制内容
