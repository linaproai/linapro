## ADDED Requirements

### Requirement: 源码插件清单发现必须容忍工作区缺失或为空

系统 SHALL 在源码插件工作区不存在或为空目录时将源码插件清单发现结果视为空集合。插件清单扫描不得因为 `apps/lina-plugins` 缺失而阻断宿主启动、插件列表读取或宿主测试。

#### Scenario: 源码插件工作区不存在
- **WHEN** 宿主扫描源码插件清单
- **AND** `apps/lina-plugins` 不存在
- **THEN** 扫描返回空源码插件集合
- **AND** 动态插件发现流程仍按配置继续执行
- **AND** 宿主启动和插件列表读取不失败

#### Scenario: 源码插件工作区为空
- **WHEN** 宿主扫描源码插件清单
- **AND** `apps/lina-plugins` 存在但不包含任何 `plugin.yaml`
- **THEN** 扫描返回空源码插件集合
- **AND** 不把空目录视为损坏的插件工作区

#### Scenario: 源码插件工作区结构无效
- **WHEN** `apps/lina-plugins` 包含目录但其中某个源码插件清单结构无效
- **THEN** 显式同步和插件治理检查必须报告该插件的结构错误
- **AND** 错误包含插件路径和待修复文件

### Requirement: 显式源码插件操作必须要求可用工作区

当调用方显式操作某个源码插件或要求完整官方插件验证时，系统 SHALL 验证官方插件工作区存在且结构有效。缺失时必须返回可操作错误，而不是静默降级为空集合。

#### Scenario: 显式同步缺失的官方插件工作区
- **WHEN** 调用方执行要求同步官方源码插件的命令
- **AND** `apps/lina-plugins` 不存在或为空
- **THEN** 命令返回明确错误
- **AND** 错误提示初始化官方插件 submodule

#### Scenario: 读取插件列表不要求官方插件工作区
- **WHEN** 管理员读取插件列表
- **AND** `apps/lina-plugins` 不存在或为空
- **THEN** 列表 API 返回动态插件和已有治理投影中仍有效的数据
- **AND** 新的源码插件发现集合为空
