## Why

LinaPro 管理后台在完成基础业务模块（用户、部门、岗位、字典、通知公告等）后，缺少系统治理层面的核心能力：管理员无法追踪用户操作行为和登录历史，无法查看在线用户和服务器运行状态，无法浏览后端 API 文档和系统运行信息，也无法在运行时动态调整系统参数。本次变更围绕系统可观测性、审计安全、自我描述和运行时配置四大维度，补齐管理后台的系统治理能力。

## What Changes

### 审计日志体系
- 新增**操作日志**模块：通过中间件自动拦截并记录所有写操作（POST/PUT/DELETE）及特殊标记的查询操作（如导出），记录操作人、操作时间、操作类型、请求参数、响应结果、耗时等信息
- 新增**登录日志**模块：在用户登录/登出时自动记录登录行为，包括登录用户名、登录时间、IP、浏览器、操作系统、登录结果等信息
- 两类日志均支持按条件筛选、查看详情、按时间范围清理（硬删除）、批量删除、导出为 xlsx 格式
- 操作日志记录机制采用中间件自动拦截 + API 定义标签的混合方案，请求参数和响应结果进行长度截断和密码脱敏

### 系统可观测性
- 新增**在线用户**功能：基于 MySQL MEMORY 引擎实现会话跟踪，展示当前在线用户列表，支持管理员强制下线指定用户
- 新增**服务监控**功能：通过 gopsutil 定时采集服务器指标（CPU、内存、磁盘、网络流量），写入数据库，支持多节点分布式部署场景；前端展示服务器信息、各项指标和 Go 运行时信息
- 改造现有认证中间件：登录时创建会话记录，登出时删除会话记录，请求校验时同时检查会话是否有效（支持强制下线和超时清理）

### 系统自我描述
- 新增**系统接口**页面：集成 Stoplight Elements OpenAPI 文档 UI，通过 iframe 嵌入静态文档页面，展示后端 API 文档并支持在线测试
- 新增**版本信息**页面：展示项目介绍、后端/前端技术组件列表，后端提供系统运行时信息 API
- 新增**组件演示**页面：通过 iframe 嵌入 vben5 官网演示页面，含加载失败处理

### 运行时配置管理
- 新增**参数设置**模块：`sys_config` 表存储键值对形式的系统参数，支持完整 CRUD、按键名查询、Excel 导出/导入（含覆盖/忽略模式）
- 优化**字典管理**导出导入功能：支持合并导出导入字典类型和字典数据到双 Sheet Excel 文件，简化操作流程

## Capabilities

### New Capabilities
- `oper-log`: 操作日志自动记录（中间件拦截）、列表查询、详情查看、按时间范围清理、批量删除、导出
- `login-log`: 登录日志自动记录、列表查询、详情查看、按时间范围清理、批量删除、导出
- `online-user`: 在线用户管理，包括会话跟踪（MySQL MEMORY 引擎）、在线用户列表查询、强制下线、不活跃会话自动清理
- `server-monitor`: 服务监控，包括定时采集服务器指标（CPU、内存、磁盘、网络、Go 运行时）、多节点数据写入数据库、前端指标展示与节点切换
- `system-api-docs`: 系统接口文档页面，集成 Stoplight Elements 展示 OpenAPI 文档并支持在线测试
- `system-info`: 版本信息页面，展示项目信息、后端/前端组件信息，后端提供系统运行时信息 API
- `component-demo`: 组件演示页面，iframe 嵌入 vben5 官网演示，含加载失败处理
- `config-management`: 系统参数设置的完整 CRUD 管理能力，包括列表查询、新增、编辑、删除、按键名查询、Excel 导出/导入
- `dict-import`: 字典类型和字典数据的导入功能，以及合并导出导入优化

### Modified Capabilities
- `user-auth`: 认证流程改造——登录时写入登录日志并创建会话记录，登出时写入登录日志并删除会话记录，中间件增加会话有效性校验以支持强制下线

## Impact

- **数据库**：新增 `sys_oper_log`、`sys_login_log`、`sys_online_session`（MEMORY 引擎）、`sys_server_monitor`、`sys_config` 五张表；新增系统监控菜单和字典数据
- **后端 API**：新增操作日志、登录日志、在线用户、服务监控、系统信息、参数设置等 RESTful 接口
- **后端服务**：新增 operlog、loginlog、online-user、server-monitor、sysinfo、sysconfig 等服务模块；新增操作日志中间件；改造认证服务和中间件
- **后端依赖**：新增 `github.com/shirou/gopsutil/v4`（系统指标采集）、`mssola/useragent`（UA 解析）
- **前端路由**：新增"系统监控"菜单（操作日志、登录日志、在线用户、服务监控）和"系统信息"菜单（系统接口、版本信息、组件演示）、参数设置菜单项
- **前端视图**：新增 monitor/ 和 about/ 目录下的页面组件、system/config/ 参数设置页面
- **前端依赖**：引入 `@scalar/api-reference`（OpenAPI 文档渲染）
- **字典数据**：新增 `sys_oper_type`（操作类型）、`sys_login_status`（登录状态）等字典类型
