## ADDED Requirements

### Requirement: 版本信息页面展示
系统 SHALL 提供一个"版本信息"页面，展示三个信息区块：关于项目、后端组件、前端组件。页面样式参考 vben5 框架的 About 页面，使用 Card + Descriptions 布局。

#### Scenario: 正常加载版本信息页面
- **WHEN** 用户点击"系统信息 > 版本信息"菜单
- **THEN** 页面展示三个卡片区块，分别显示项目介绍、后端技术组件列表、前端技术组件列表

### Requirement: 后端系统信息 API
系统 SHALL 提供 `GET /api/v1/system/info` 接口，返回系统运行时信息。该接口 MUST 在鉴权路由组内，仅登录用户可访问。

#### Scenario: 获取系统运行时信息
- **WHEN** 前端请求 `GET /api/v1/system/info`
- **THEN** 接口返回包含以下字段的 JSON 数据：Go 版本（运行时动态获取）、GoFrame 版本（运行时动态获取）、操作系统及架构、数据库版本、系统启动时间、系统运行时长、前后端组件信息列表

### Requirement: 关于项目区块
关于项目区块 SHALL 展示项目名称、项目介绍、当前版本号、开源许可证、项目主页链接。这些信息在前端配置对象中定义。项目名称与项目介绍放在第一行，其他信息放在第二行。

#### Scenario: 展示项目基本信息
- **WHEN** 版本信息页面加载完成
- **THEN** "关于项目"区块显示项目名称、项目介绍、版本号、许可证类型，项目主页为可点击的外链

### Requirement: 后端组件区块
后端组件区块 SHALL 展示后端使用的技术组件列表，每个组件包含名称、版本号和官网链接。组件列表从后端 API 获取。

#### Scenario: 展示后端组件列表
- **WHEN** 版本信息页面加载完成
- **THEN** "后端组件"区块以网格布局展示 GoFrame、MySQL、JWT 等后端组件，每个组件的名称和版本号可见，官网链接可点击跳转

### Requirement: 前端组件区块
前端组件区块 SHALL 展示前端使用的技术组件列表，每个组件包含名称、版本号和官网链接。组件列表从后端 API 获取。

#### Scenario: 展示前端组件列表
- **WHEN** 版本信息页面加载完成
- **THEN** "前端组件"区块以网格布局展示 Vue、Vben5、Ant Design Vue、TypeScript 等前端组件，每个组件的名称和版本号可见，官网链接可点击跳转

### Requirement: 外链地址配置化
所有第三方组件的官网链接 SHALL 在后端配置文件中集中定义，通过 API 返回给前端，修改链接时无需改动页面组件代码。

#### Scenario: 修改外链地址
- **WHEN** 开发者修改后端配置文件中某个组件的链接地址
- **THEN** 版本信息页面对应组件的链接自动更新为新地址
