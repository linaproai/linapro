# 修复接口文档请求地址前缀

当前系统接口文档页面通过 Stoplight 渲染后端 `/api.json`，但 OpenAPI `servers` 地址来自后端元数据中的固定 `http://localhost:8080`。当服务部署在容器端口映射、反向代理或直接通过宿主机映射端口访问时，接口文档中每个接口展示和 Try It 使用的请求地址会与浏览器实际可访问地址不一致。

本变更将接口文档请求地址前缀改为按 `/api.json` 当前请求动态生成：前端开发服务代理访问时由后端看到的 `8080` 地址生成；后端直接暴露访问时与浏览器访问后端的 scheme、host 和端口保持一致。元数据继续负责标题、描述、版本和服务描述，不再作为运行时请求地址的权威来源。

## 范围

- 修改宿主 `/api.json` 处理链路，按请求 origin 设置 OpenAPI `servers[0].url`。
- 保持接口文档页面继续加载相对路径 `/api.json`，以适配前端代理和后端直连访问。
- 增加可复现的后端单元测试与接口文档 E2E 覆盖。

## 非目标

- 不新增 OpenAPI 配置项。
- 不调整 API DTO 文档源文本、接口路径、鉴权逻辑或 Stoplight 核心渲染库。
- 不修改数据库结构或初始化 SQL。
