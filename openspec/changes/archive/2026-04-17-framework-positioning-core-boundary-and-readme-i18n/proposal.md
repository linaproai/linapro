## Why

当前仓库在产品定位、系统文案、接口叙事和目录说明上仍大量沿用“后台管理系统”视角，这与 `LinaPro` 实际已经具备的插件宿主、动态扩展、AI 协作研发流程和全栈工程底座能力不一致。继续维持这套表述会让后续前后端实现、文档编写和 OpenSpec 产物持续朝“页面驱动的后台项目”收缩，而不是围绕框架能力演进。

同时，`apps/lina-core` 作为宿主核心服务，已经开始承载部分面向具体前端工作台的路由、树选择器和聚合响应。如果不尽快把“核心通用能力”与“工作台适配”分层收敛，后续后台页面调整会反向驱动核心接口变更，增加耦合和维护成本。仓库内 `README` 也缺少统一的国际化规则，无法支撑后续多语言项目协作和对外呈现。

## What Changes

- 统一将 `LinaPro` 的项目定位修正为“`AI驱动的全栈开发框架`”，调整前端、后端、配置、脚本输出、系统信息页、OpenAPI 文案和仓库说明中的不合适描述，弱化“仅一个后台管理系统”的叙事。
- 新增顶级规范，明确 `apps/lina-core` 是框架核心宿主服务，设计优先保证通用模块接口能力、组件能力、系统治理能力和插件扩展能力，不得与具体管理后台页面需求强绑定。
- 新增工作台适配约束，要求路由装配、工作台聚合、树选择器、下拉选项等页面导向输出显式归类为工作台适配接口，不得直接污染 `lina-core` 的通用领域契约。
- 建立全仓库 `README` 国际化规范：所有目录级主说明文档使用英文 `README.md`，并同步提供内容一致的中文镜像 `README.zh_CN.md`；后续新增说明文档亦遵循同一规则。
- 为仓库根目录补齐英文主 README 与中文镜像 README，并把现有子目录 README 的渐进式双语治理纳入统一任务清单。
- 调整活跃 OpenSpec 主规范和项目说明规则，使后续 proposal、design、spec、tasks 与 README 编写继续沿用新的定位和文档约束。
- 为宿主 `init` / `mock` 敏感数据库启动命令补齐显式确认护栏，要求 `Makefile` 与后端命令实现同时校验确认参数，并在任一 `SQL` 失败时首错即停。
- 统一源码插件与动态插件的安装审查入口：安装前先展示单一详情确认窗口，动态插件在安装阶段固化授权快照，后续启用优先复用已确认快照。
- 收敛默认工作台首页与分析页的模板化演示内容，改为 LinaPro 当前真实可达的导航入口、运维语义与清晰的指标时间范围表达。
- 将仓库根 `make dev` 产生的开发日志从系统 `/tmp` 收敛到项目根 `temp/` 目录，便于在仓库内集中定位开发期运行产物。

## Capabilities

### New Capabilities
- `project-positioning-governance`: 统一项目定位、系统元数据和用户可见说明文案的治理要求，确保 `LinaPro` 对内对外均以“AI驱动的全栈开发框架”表述。
- `core-host-boundary-governance`: 规范 `lina-core` 的核心宿主边界与工作台适配边界，约束页面需求不得直接驱动核心通用接口和能力模型变更。
- `readme-localization-governance`: 规范全仓库 `README.md`/`README.zh_CN.md` 的双语镜像要求、同步维护要求和新增文档约束。
- `database-bootstrap-commands`: 规范宿主数据库初始化与演示数据加载命令的确认护栏、失败语义和可观测反馈。

### Modified Capabilities
- `base-layout`: 将宿主统一布局的表述从“后台基础布局”收敛为默认管理工作台语义，避免把项目整体定位绑定到后台系统。
- `plugin-ui-integration`: 将插件页面接入场景中的“后台导航”“后台内页环境”等表述收敛为宿主导航和宿主工作区环境语义，保持能力表述与框架定位一致，并同步要求默认工作台首页只暴露真实可达入口。
- `system-info`: 调整系统信息页关于项目、组件清单与项目介绍的要求，使页面展示与新的项目定位、技术栈叙事和框架能力边界保持一致。
- `system-api-docs`: 调整系统接口文档页面和 OpenAPI 元数据要求，使标题、说明和页面文案不再将宿主 API 定义为“后台管理系统接口”。
- `plugin-manifest-lifecycle`: 统一插件安装前的详情审查入口，并要求动态插件 release 复用安装阶段确认的授权快照。
- `plugin-permission-governance`: 要求插件自有资源接口按插件资源权限校验访问，不得额外要求插件管理后台治理权限。
- `dashboard-workbench`: 收敛默认工作台首页与分析页的模板演示内容，改为真实导航、运维语义和可切换的指标时间范围视图。

## Impact

- 受影响文档与规范：`CLAUDE.md`、`openspec/config.yaml`、相关 OpenSpec 主规范、仓库根 README 及现有子目录 README 集合。
- 受影响前端文案与元数据：登录页国际化文案、系统信息页项目介绍、前端入口 HTML meta、插件 Slot 描述、相关 E2E 断言。
- 受影响后端与配置元数据：`OpenAPI` 标题与说明、系统组件描述、打包配置模板、脚本状态输出等项目级元信息。
- 受影响实现边界：现有面向工作台的菜单路由、当前用户启动数据、树选择器/选项投影等接口与注释需要按“核心能力 vs 工作台适配”重新归类和命名。
- 受影响命令与脚本入口：仓库根 `Makefile`、`apps/lina-core/Makefile`、`apps/lina-core/internal/cmd/` 下 `init` / `mock` 命令及其公共 `SQL` 执行流程。
- 受影响插件治理与工作台页面：插件管理页安装链路、动态插件授权展示、默认工作台首页与分析页，以及相关插件权限回归用例。
- 受影响开发期运行产物：仓库根 `make dev` 的日志输出路径、状态提示与 `temp/` 目录下的临时日志文件。

## Merged Scope

为恢复仓库“单一活跃迭代统一管理”的状态，以下 3 个活跃变更并入当前 `framework-positioning-core-boundary-and-readme-i18n` 迭代，不再保留独立活跃目录。

### `guard-db-init-and-mock-commands`

- 为宿主 `init` / `mock` 建立显式确认护栏，要求根目录与 `apps/lina-core` 的 `make` 入口都显式传入对应 `confirm` 值。
- 后端命令实现同步执行最终确认校验，避免直接调用 `go run main.go init/mock` 绕过安全限制。
- 公共 `SQL` 执行语义改为首错即停并返回失败状态，保证数据库启动命令失败可观测。

### `plugin-install-authorization-dialog-optimization`

- 源码插件与动态插件安装前统一通过单一安装审查弹窗查看详情后再确认。
- 动态插件在安装阶段确认宿主服务授权快照，后续启用优先复用该快照，不再重复弹窗。
- 默认工作台首页与分析页移除模板演示叙事，改为 LinaPro 当前真实可达的入口、运维信息与指标分区。
- 插件自有资源接口权限校验收敛到插件资源权限本身，不再额外要求 `plugin:query` 这类治理权限。

### `redirect-dev-logs-to-project-temp`

- 将根目录 `make dev` 生成的前后端开发日志从 `/tmp` 调整到仓库根 `temp/`。
- 同步收敛开发状态提示与运行时产物目录，方便在仓库内直接定位开发日志。
