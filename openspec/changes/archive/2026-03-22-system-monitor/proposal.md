## Why

管理后台缺少系统运行时监控能力。管理员无法查看当前在线用户、无法强制下线异常会话，也无法实时了解服务器的 CPU、内存、磁盘、网络等运行状态。v0.7.0 迭代新增系统监控模块，提供在线用户管理和服务器指标监控两大功能，提升系统的可观测性和运维效率。

## What Changes

- 新增**在线用户**功能：基于 MySQL MEMORY 引擎实现会话跟踪，展示当前在线用户列表（用户信息、登录时间、IP、浏览器、操作系统等），支持管理员强制下线指定用户
- 新增**服务监控**功能：通过 gopsutil 定时采集服务器指标（CPU、内存、磁盘、网络流量），写入数据库，支持多节点分布式部署场景；前端展示服务器信息、各项指标和 Go 运行时信息
- 新增**系统监控菜单模块**：在导航菜单中新增"系统监控"一级菜单，包含"在线用户"和"服务监控"两个子菜单
- 改造现有认证中间件：登录时创建会话记录，登出时删除会话记录，请求校验时同时检查会话是否有效（支持强制下线）

## Capabilities

### New Capabilities

- `online-user`: 在线用户管理，包括会话跟踪（MySQL MEMORY 引擎）、在线用户列表查询、强制下线功能，以及会话存储抽象层设计
- `server-monitor`: 服务监控，包括定时采集服务器指标（CPU、内存、磁盘、网络、Go 运行时）、多节点数据写入数据库、前端指标展示与节点切换

### Modified Capabilities

- `user-auth`: 认证流程改造——登录时写入会话记录，登出时删除会话记录，中间件增加会话有效性校验以支持强制下线

## Impact

- **后端新增依赖**: `github.com/shirou/gopsutil/v4`（系统指标采集）
- **数据库变更**: 新增 `sys_online_session`（MEMORY 引擎）和 `sys_server_monitor` 表；新增系统监控相关菜单和字典数据
- **API 新增**: `GET /monitor/online/list`、`DELETE /monitor/online/{tokenId}`、`GET /monitor/server`
- **认证中间件改造**: 登录/登出流程增加会话管理，请求校验增加会话存在性检查
- **前端新增**: 系统监控路由模块、在线用户页面、服务监控页面及相关 API 调用
