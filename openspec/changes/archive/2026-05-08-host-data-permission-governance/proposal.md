## Why

角色管理已经提供 `dataScope`（全部数据 / 本部门数据 / 仅本人数据）配置，但宿主核心接口目前主要只校验菜单和按钮权限，尚未把数据范围稳定应用到宿主业务数据的查询、详情、导出、下载、更新、删除和执行类操作中。这会导致角色表单中的“数据权限”语义与宿主实际访问控制不一致。

本变更建立宿主侧统一数据权限治理能力，并分析、划分哪些宿主业务模块适合纳入数据权限控制，避免把系统治理数据（菜单、角色、配置、字典、插件治理等）错误地按部门或本人范围过滤。

## What Changes

- 新增宿主数据权限治理能力：统一解析当前用户有效数据范围，支持超级管理员绕过、多角色最宽范围合并、无用户上下文失败关闭。
- 新增宿主资源数据权限策略注册：每个适用资源显式声明按用户列、部门列或组织能力解析的过滤方式，禁止接口临时手写分散规则。
- 首批接入适合数据权限管理的宿主业务模块：
  - 用户管理：按用户归属部门和用户自身限制列表、详情、导出、更新、删除、状态变更、重置密码、角色授权用户列表等操作。
  - 文件管理：按上传人 `created_by` 和上传人所属部门限制列表、详情、批量信息、按 ID 下载、删除、聚合选项；上传文件 URL 访问 `/api/v1/uploads/*path` 暂时作为公开浏览器访问入口，仅保留路径和元数据校验。
  - 定时任务管理：仅对用户创建任务（`is_builtin=0`）按 `created_by` 和创建人所属部门限制列表、详情、编辑、删除、启停、触发及相关日志操作；内置任务投影保持系统治理语义。
  - 在线用户：按会话 `user_id` 与用户所属部门限制在线会话列表和强制下线。
  - 用户消息：保持当前用户自隔离语义，明确不得因角色拥有“全部数据权限”而读取他人消息。
- 明确不纳入数据权限过滤的宿主治理模块：菜单、角色基础列表与详情、字典、配置、插件治理、i18n、健康检查、系统信息、公共配置、缓存、锁和集群协调等，继续仅由功能权限与系统保护规则治理。
- 新增数据权限边界测试：覆盖列表、详情、导出、下载、写操作、跨部门拒绝、无组织能力降级、超级管理员绕过和多角色最宽范围合并。

## Capabilities

### New Capabilities

- `host-data-permission-governance`：定义宿主数据权限解析、资源策略注册、模块适用性分类、查询注入、写操作边界校验、缓存一致性与首批宿主资源接入规则。

### Modified Capabilities

- `role-management`：角色数据范围不再只是存储字段，必须作为宿主数据权限解析的输入，并在角色数据范围变更后触发访问拓扑缓存失效。
- `user-management`：用户管理中的用户数据查询和变更必须遵循当前调用方的数据权限范围。
- `online-user`：在线会话列表和强制下线能力必须遵循当前调用方的数据权限范围。
- `cron-job-management`：用户创建的定时任务与关联执行日志必须遵循当前调用方的数据权限范围，内置任务投影保持系统治理语义。
- `user-message`：用户消息保持当前用户自隔离边界，数据权限不得放宽该边界。

## Impact

- 受影响后端代码：
  - `apps/lina-core/internal/service/role/`：解析有效数据范围、访问上下文或新数据权限服务接入、缓存失效联动。
  - `apps/lina-core/internal/service/user/`：用户资源列表、详情、导出、写操作和角色授权用户列表接入数据权限策略。
  - `apps/lina-core/internal/service/file/`：文件列表、详情、批量信息、按 ID 下载、删除、聚合选项接入上传人范围过滤；上传文件 URL 公开访问保留路径和元数据校验。
  - `apps/lina-core/internal/service/jobmgmt/`、`joblog` 相关控制器：用户创建任务与日志接入创建人范围过滤。
  - `apps/lina-core/internal/service/session/` 或在线用户投影服务：在线会话列表和强制下线接入用户范围过滤。
  - `apps/lina-core/internal/service/orgcap/` 与 `apps/lina-core/pkg/orgcap/`：补充部门范围解析所需的稳定组织能力接口（如按部门集合解析直接归属用户）。
- 受影响前端代码：
  - 角色管理无需新增可见字段；保留现有 `dataScope` 表单。
  - 如错误提示或空状态文案发生变化，需要同步维护运行时 i18n 资源。
- 受影响测试：
  - 新增后端单元测试和集成测试覆盖数据权限解析与模块策略。
  - 对涉及前端交互的用户、文件、定时任务、在线用户页面补充 E2E 用例，TC 编号遵循 `lina-e2e` 规范。
- i18n 影响：
  - 本变更不新增角色表单字段。
  - 若新增数据权限相关 `bizerr` 错误码，必须同步维护 `zh-CN`、`en-US`、`zh-TW` 运行时错误翻译。
  - 本变更调整上传文件 URL 访问接口文档时，需要同步维护非英文 apidoc 资源；其他未修改 API DTO 文档元数据的路径不新增 apidoc i18n。
- 缓存一致性影响：
  - 角色 `dataScope`、用户角色关系和组织归属变化会影响数据权限结果，必须复用现有访问拓扑或新增明确作用域的缓存修订号。
  - 不允许普通业务路径无理由清空所有缓存；所有失效必须带明确权限或组织数据作用域。
