## MODIFIED Requirements

### Requirement: 全局字典组件和 Store 必须缓存字典数据

系统 SHALL 提供可复用的 `DictTag` 组件和 Pinia 字典 Store，使字典选项加载一次后跨页面复用，并在字典数据变更后同步已挂载的字典标签。

#### Scenario: DictTag 渲染字典值

- **当** `DictTag` 接收字典选项和值时
- **则** 显示匹配的标签和配置的标签样式
- **且** 不匹配的值显示回退文本

#### Scenario: 字典 Store 请求去重

- **当** 多个组件并发请求同一字典类型时
- **则** 仅发出一个 API 请求
- **且** 所有调用方共享同一结果

#### Scenario: 字典缓存刷新

- **当** 用户刷新字典缓存时
- **则** 缓存的字典数据被清除，后续请求从 API 重新加载

#### Scenario: 字典数据变更后已挂载标签同步刷新

- **当** 用户修改某个字典类型下字典数据的标签文字、标签样式或 CSS 类
- **则** 字典 Store 按该字典类型从 API 重新加载选项
- **且** 已经挂载并使用该字典类型的列表标签同步显示最新字典内容
- **且** 刷新过程不得依赖重新打开页面或重新挂载列表组件

## ADDED Requirements

### Requirement: 工作台 Tab 切换必须保留列表页局部状态

默认管理工作台 SHALL 对内建可分页列表页面启用 Tab 页面缓存，使用户在不同 Tab 间切换时保留当前页码、分页大小和页面局部状态。

#### Scenario: 切换 Tab 后保留分页状态

- **当** 用户在内建列表页面切换到非第一页
- **且** 用户打开或切换到另一个工作台 Tab
- **且** 用户切回原列表页 Tab
- **则** 原列表页仍显示切换前的页码和分页大小
- **且** 系统不会因 Tab 切换重新挂载该列表页并重置分页
