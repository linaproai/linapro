## Context

本变更涉及 `linapro-org-core` 源码插件前端适配层和宿主内建能力的三组问题修复，以及动态插件样例 CI 稳定性纠正。所有修复均限于前端缓存语义、菜单 Seed 配置和测试 fixture 准备，不改变核心宿主边界、后端 API 契约或数据库结构。

## Goals / Non-Goals

**Goals**

- 使部门管理页面在部门树为空时能创建首个顶级部门。
- 使字典标签更新后已打开列表实时同步显示最新标签。
- 使工作台 Tab 切换后列表页保留分页状态。
- 使动态插件样例在 CI 干净 checkout 中单测稳定通过，同时保持 `config/config.yaml` 运行配置契约。

**Non-Goals**

- 不修改 `apps/lina-core` 核心领域契约、通用 service 语义或存储模型。
- 不新增补丁 SQL 或兼容性迁移脚本。
- 不引入前端路由表硬编码兜底。
- 不扩展公共抽象层。

## Decisions

### 部门空树创建

部门后端已将 `parentId=0` 定义为顶级部门语义。修复方案是在部门新增抽屉的上级部门 TreeSelect 中合成本地"顶级部门"节点，工具栏新增场景默认提交 `parentId=0`。真实部门树仍来自现有 `dept/tree` 和 `dept/exclude/{id}` 接口，前端只做展示候选归一化。为避免扁平 DTO 与树节点 DTO 在同一控件中显示不一致，抽屉本地将候选数据统一投影为 `id`、`label`、`children` 结构，该归一化只作用于当前页面控件。

### 字典缓存同步

字典 Store 保持按字典类型缓存选项数组，但新增按类型强制刷新能力。刷新指定字典类型时，不替换数组对象，而是对原数组执行就地 splice，让已挂载组件继续持有同一个响应式数组引用并立即看到最新标签。`resetCache()` 保留给登录态重置、语言切换和未知范围刷新等全局场景。字典数据新增、更新和删除应优先按当前 dictType 精确刷新，导入等未知影响范围的路径刷新全部已缓存字典类型。刷新失败保留旧数组并允许后续重试，避免写入成功后的页面流程因缓存刷新失败中断。

### Tab 分页状态保持

默认工作台 Tab 缓存的权威数据源是后端菜单投影：`sys_menu.is_cache` 通过 `/menus/all` 投影为 `meta.keepAlive`，前端 tabbar 只缓存 `meta.keepAlive=true` 的路由。修复方式是直接修正宿主原始菜单 seed，将内建可分页 routed 菜单的 `is_cache` 设置为 `1`，不在前端路由表额外硬编码兜底，也不保留面向旧库的补丁 SQL。

### 动态插件样例配置契约

动态插件样例的 `plugin.yaml` 和测试断言要求打包 `manifest/config/config.yaml`。该文件按插件配置规范是实际运行配置，默认被 `.gitignore` 忽略。正确修复是保持样例契约继续使用 `config/config.yaml`，并由单测在构建前从 `config.example.yaml` 准备临时 `config.yaml` fixture。

## Risks / Trade-offs

- 字典缓存刷新失败时保留旧数组并允许后续重试，这意味着极端情况下用户可能短暂看到旧标签，但避免了写入成功后的页面流程中断，属于可接受的权衡。
- 直接修正宿主 Seed DML 而非新增补丁 SQL，适用于全新项目无历史负担的定位，但要求重新执行 `make db.init` 才能生效。
- 部门候选归一化逻辑仅作用于当前页面控件，未扩展为公共抽象，保持了实现简洁性但可能在未来需要类似处理时产生重复代码。
