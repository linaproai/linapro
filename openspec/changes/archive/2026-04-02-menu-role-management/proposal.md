## Why

当前 Lina 项目缺少菜单管理和角色管理功能，导致系统无法实现精细化的权限控制。用户登录后无法根据角色动态展示菜单，也无法对不同角色分配不同的功能权限。这是后台管理系统的核心基础功能，需要尽早实现以支撑后续业务功能的权限隔离。

## What Changes

- **新增菜单管理模块**：实现系统菜单的增删改查功能，支持树形层级结构（目录/菜单/按钮三种类型），支持外链、缓存、i18n 国际化等功能
- **新增角色管理模块**：实现角色的增删改查功能，支持角色与菜单的关联分配，支持为角色分配用户
- **扩展用户管理模块**：用户创建/编辑时支持选择关联角色，用户列表展示关联的角色信息
- **扩展登录认证流程**：登录后根据用户角色返回对应的菜单树，前端基于菜单树控制页面访问和菜单展示
- **简化数据权限**：实现三种数据权限范围（全部/本部门/仅本人），后续可扩展

## Capabilities

### New Capabilities

- `menu-management`: 系统菜单管理功能，支持树形层级结构、目录/菜单/按钮三种类型、外链、缓存、状态控制等
- `role-management`: 系统角色管理功能，支持角色的增删改查、菜单权限分配、用户分配，数据权限范围设置
- `user-role-association`: 用户与角色的关联功能，支持在用户管理中选择角色、展示角色信息

### Modified Capabilities

- `user-auth`: 扩展登录认证流程，返回用户角色和菜单树信息
- `user-management`: 扩展用户管理，增加角色关联功能

## Impact

### 后端影响

- 新增数据库表：`sys_menu`、`sys_role`、`sys_role_menu`、`sys_user_role`
- 新增 API 模块：`menu`、`role`
- 修改 API 模块：`auth`（返回菜单树）、`user`（增加角色关联）
- 新增字典类型：菜单状态、显示状态等

### 前端影响

- 新增页面：菜单管理、角色管理、角色用户分配
- 修改页面：用户管理（增加角色选择和展示）
- 修改登录流程：基于菜单树动态生成路由

### 权限控制

- 登录后根据用户角色获取对应菜单，前端基于菜单控制页面访问
- 按钮级权限通过菜单中的按钮类型（B）控制
