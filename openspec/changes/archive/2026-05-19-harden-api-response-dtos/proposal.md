## Why

当前部分后端 API 响应直接嵌入数据库实体，例如用户响应嵌入 `entity.SysUser`，文件响应嵌入 `entity.SysFile`。这会让 API 合同被数据库结构被动驱动，并可能把密码字段、软删除字段、存储路径、文件哈希、租户内部字段等实现细节暴露给调用端。即使个别服务查询当前已经排除了敏感字段，响应 DTO 仍然包含对应 JSON 字段，后续实体新增字段时也容易被无意暴露。

## What Changes

- 将宿主 API 响应中的 `entity.*` 嵌入和 `[]*entity.*` 列表替换为显式响应 DTO。
- 优先覆盖用户、文件、系统配置、字典、定时任务、任务日志和任务分组这些已发现直接暴露实体的 API。
- 控制器在响应边界显式映射允许暴露的字段，避免把实体指针直接传入 API 响应。
- 增加自动化测试或静态验证，确保 API DTO 不再依赖 `internal/model/entity`，并验证用户响应不包含 `password` 等内部字段。

## Impact

- 影响宿主后端 API 响应结构定义与控制器响应组装。
- 不新增或修改数据库 schema、DAO/DO/Entity 生成文件。
- 不改变 REST 路径、HTTP 方法、权限点或数据权限查询逻辑。
- 同步清理文件详情页运行时翻译和宿主 apidoc i18n 中已经不再暴露的响应字段翻译。
- 不新增运行时缓存、缓存失效或跨实例协调逻辑。
