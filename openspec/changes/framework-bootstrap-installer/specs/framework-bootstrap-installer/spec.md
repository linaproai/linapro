## ADDED Requirements

### Requirement: 安装入口必须提供跨平台的一致能力
系统 SHALL 在仓库根 `hack/scripts/install/` 下提供面向整个 `LinaPro` 仓库的快速安装脚本入口，其中 `install.sh` 用于 `macOS/Linux`，`install.ps1` 用于 `Windows PowerShell`。两个入口 MUST 共享一致的核心参数语义，至少覆盖仓库来源、源码引用、目标目录模式与覆盖保护等安装行为，而不是分别实现互不兼容的交互模型。

#### Scenario: Unix-like 用户通过 Shell 入口执行安装
- **WHEN** 用户在 `macOS` 或 `Linux` 环境执行 `hack/scripts/install/install.sh`
- **THEN** 脚本能够解析与快速安装相关的核心参数
- **AND** 脚本开始面向整个 `LinaPro` 仓库执行源码下载与落地流程

#### Scenario: Windows 用户通过 PowerShell 入口执行安装
- **WHEN** 用户在 `Windows PowerShell` 环境执行 `hack/scripts/install/install.ps1`
- **THEN** 脚本能够解析与 Shell 入口等价的核心安装参数
- **AND** 脚本开始面向整个 `LinaPro` 仓库执行源码下载与落地流程

### Requirement: 安装流程必须基于源码归档下载而不是 Git 克隆
系统 SHALL 支持根据用户指定的仓库和 `ref` 下载 `LinaPro` 源码归档，并使用归档内容完成本地项目落地。安装主流程 MUST 不依赖 `git clone` 或本机预装 Git；在 `macOS/Linux` 下应优先使用适合 Shell 环境的归档格式，在 `Windows` 下应优先使用适合 PowerShell 原生解压的归档格式。

#### Scenario: 使用默认仓库与默认引用下载源码归档
- **WHEN** 用户未显式指定仓库或 `ref` 参数执行安装脚本
- **THEN** 脚本下载官方默认仓库的默认引用源码归档
- **AND** 默认引用优先解析为仓库当前最新的稳定语义化标签版本
- **AND** 本地安装流程不要求用户预先安装 Git

#### Scenario: 使用指定仓库与指定引用下载源码归档
- **WHEN** 用户在安装脚本中显式传入仓库地址与 `ref`
- **THEN** 脚本从该仓库的对应引用下载源码归档
- **AND** 随后使用该归档内容执行解压与本地目录落地

#### Scenario: 默认稳定标签缺失时回退到主分支
- **WHEN** 用户未显式指定 `ref`，且目标仓库不存在可识别的稳定语义化标签版本
- **THEN** 脚本回退到约定的默认主分支引用继续执行源码归档下载
- **AND** 输出中明确展示最终解析得到的引用值

### Requirement: 安装脚本必须安全地处理当前目录与指定目录模式
系统 SHALL 支持将源码解压到当前目录或显式指定目录，并在默认情况下避免把源码直接覆盖到非空目录。脚本 MUST 先在临时目录完成归档解压与顶层目录识别，再将实际项目文件移动到最终目标位置；当目标目录非空且用户未显式允许覆盖时，脚本 MUST 拒绝继续执行。

#### Scenario: 用户显式要求落地到当前目录
- **WHEN** 用户传入当前目录模式参数执行安装脚本
- **THEN** 脚本将解压后的项目内容落地到当前工作目录
- **AND** 若当前目录已存在非安装脚本创建的内容且未显式允许覆盖，脚本拒绝继续执行

#### Scenario: 用户显式要求落地到指定目录
- **WHEN** 用户传入目标目录参数执行安装脚本
- **THEN** 脚本将解压后的项目内容落地到指定目录
- **AND** 若目标目录不存在，脚本创建该目录或其父目录后再完成安装

### Requirement: 安装完成后必须输出环境体检与下一步引导
系统 SHALL 在源码落地完成后，检查运行 `LinaPro` 开发流程所需的关键依赖，并输出清晰的安装后引导。体检结果 MUST 至少覆盖 `Go`、`Node.js`、`pnpm`、`MySQL`、`make` 的存在性或版本信息；安装成功输出 MUST 告知用户项目路径以及后续推荐执行的关键命令。

#### Scenario: 关键依赖齐全时输出可继续启动的引导
- **WHEN** 安装脚本完成源码落地且检测到关键依赖均已满足
- **THEN** 脚本输出项目所在目录
- **AND** 脚本提示用户继续执行如 `make init`、`make mock`、`make dev` 等后续命令

#### Scenario: 存在缺失依赖时输出诊断结果
- **WHEN** 安装脚本完成源码落地但发现一个或多个关键依赖缺失
- **THEN** 脚本逐项输出缺失或不满足条件的依赖
- **AND** 脚本仍然明确告知用户项目已成功落地以及下一步补环境的方向
