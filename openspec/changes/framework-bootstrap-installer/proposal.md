## Why

当前 `LinaPro` 的本地上手流程仍然偏向“熟悉仓库结构的开发者”：用户需要先定位仓库、手动下载或克隆源码、自己决定解压目录，再分别确认本机是否具备 `Go`、`Node.js`、`pnpm`、`MySQL`、`make` 等依赖。对于首次接触框架的用户来说，环境准备路径过长且跨平台差异明显，容易在真正开始使用前就卡住。

作为”面向可持续交付的 AI 原生全栈框架”，`LinaPro` 应该提供一个统一、低门槛、跨平台的源码快速落地入口，让用户可以通过一条安装命令把指定仓库版本下载到本地目标目录，并立即得到后续初始化指引与环境检查结果。

## What Changes

- 新增仓库级“快速安装/源码引导”能力，在 `hack/scripts/install/` 下提供官方安装脚本入口，用于快速下载并落地整个 `LinaPro` 仓库源码，而不是只覆盖单个子项目。
- 为 `macOS` / `Linux` 提供 `install.sh`，为 `Windows` 提供 `install.ps1`，统一支持从指定代码仓库与指定分支、标签或提交引用下载压缩包，而不是依赖 `git clone`。
- 安装脚本默认通过 GitHub/Codeload 压缩包分发源码，优先使用更轻量的归档下载方式，并根据用户参数决定将项目解压到当前目录下的新子目录，还是解压到显式指定目录。
- 安装脚本需要内置安全目录策略：检测目标目录是否已存在、是否为空、是否允许覆盖，并在默认情况下避免把源码直接散落覆盖到非空目录中。
- 安装脚本在源码落地后输出统一的环境体检与下一步引导，包括但不限于 `Go`、`Node.js`、`pnpm`、`MySQL`、`make` 等依赖检查结果，以及 `make init`、`make dev` 等后续命令提示。
- 补充仓库根 `README.md` 与 `README.zh_CN.md` 的快速安装说明，分别给出 `macOS/Linux` 与 `Windows` 的推荐使用方式，并明确官方托管入口与仓库内脚本之间的映射关系。

## Capabilities

### New Capabilities
- `framework-bootstrap-installer`: 提供跨平台的源码快速下载、目标目录落地、安全解压、环境体检与安装后引导能力。

### Modified Capabilities
- None.

## Impact

- **仓库脚本**: 影响仓库根目录 `hack/scripts/install/`，需要新增跨平台安装脚本并约定统一参数模型。
- **项目文档**: 影响仓库根 `README.md`、`README.zh_CN.md` 的快速开始与安装说明。
- **开发体验**: 影响首次使用 `LinaPro` 的环境准备路径、目录初始化方式与跨平台 onboarding 体验。
- **分发约定**: 需要明确 GitHub/Codeload 压缩包下载规则、目标目录策略，以及官方安装入口与仓库脚本版本的一致性约束。
