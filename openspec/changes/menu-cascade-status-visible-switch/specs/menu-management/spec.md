## ADDED Requirements

### Requirement: 菜单状态与显示双向级联

系统 SHALL 在菜单状态或显示字段被写入时，将该字段的目标值同步到其全部后代菜单。停用/隐藏与启用/显示均 MUST 级联。

#### Scenario: 停用父级目录级联停用子级

- **当** 管理员将某个菜单的状态从启用改为停用
- **且** 该菜单存在一级或多级后代菜单
- **则** 系统在同一事务内将该菜单及其所有后代菜单的 status 写为停用
- **且** 不得修改后代菜单的 visible 字段（除非本次同时请求写入 visible）

#### Scenario: 启用父级目录级联启用子级

- **当** 管理员将某个菜单的状态从停用改为启用
- **且** 该菜单存在一级或多级后代菜单
- **则** 系统在同一事务内将该菜单及其所有后代菜单的 status 写为启用
- **且** 不得修改后代菜单的 visible 字段（除非本次同时请求写入 visible）

#### Scenario: 隐藏父级目录级联隐藏子级

- **当** 管理员将某个菜单的 visible 从显示改为隐藏
- **且** 该菜单存在一级或多级后代菜单
- **则** 系统在同一事务内将该菜单及其所有后代菜单的 visible 写为隐藏
- **且** 不得修改后代菜单的 status 字段（除非本次同时请求写入 status）

#### Scenario: 显示父级目录级联显示子级

- **当** 管理员将某个菜单的 visible 从隐藏改为显示
- **且** 该菜单存在一级或多级后代菜单
- **则** 系统在同一事务内将该菜单及其所有后代菜单的 visible 写为显示
- **且** 不得修改后代菜单的 status 字段（除非本次同时请求写入 status）

#### Scenario: 无子级菜单的状态或显示变更

- **当** 管理员变更一个没有后代的菜单的状态或显示
- **则** 系统仅更新该菜单自身
- **且** 操作成功完成

### Requirement: 菜单列表支持状态与显示开关

系统 SHALL 在菜单管理列表中以开关组件展示并切换「状态」与「是否显示」，允许管理员在不打开编辑抽屉的情况下快速切换。

#### Scenario: 列表开关切换状态

- **当** 管理员在菜单列表行内点击状态开关
- **则** 系统调用菜单更新接口写入目标 status
- **且** 列表刷新后该行状态与开关状态一致
- **且** 若该菜单有后代，列表中后代状态同步为目标值

#### Scenario: 列表开关切换显示

- **当** 管理员在菜单列表行内点击显示开关
- **则** 系统调用菜单更新接口写入目标 visible
- **且** 列表刷新后该行显示状态与开关状态一致
- **且** 若该菜单有后代，列表中后代显示状态同步为目标值

#### Scenario: 开关切换后刷新可访问导航

- **当** 管理员通过列表开关成功修改当前用户可见菜单的状态或显示
- **则** 当前登录用户的左侧导航在刷新链路中立即收敛到最新结果

## MODIFIED Requirements

### Requirement: 菜单状态控制

系统 SHALL 支持菜单启用/禁用状态切换。写入 status 或 visible 时 MUST 将该字段目标值级联到全部后代菜单。

#### Scenario: 禁用菜单

- **当** 用户将菜单状态更改为禁用时
- **则** 该菜单不会出现在前端菜单栏
- **且** 通过 URL 直接访问该菜单路由时，前端应拒绝访问
- **且** 其所有后代菜单也被停用，同样不会出现在前端菜单栏

#### Scenario: 启用菜单

- **当** 用户将菜单状态更改为启用时
- **则** 该菜单及其所有后代菜单同步启用
- **且** 在权限与显示条件满足时，对应菜单可重新出现在前端菜单栏

#### Scenario: 隐藏菜单

- **当** 用户将菜单的 visible 字段设置为隐藏时
- **则** 该菜单不会出现在前端菜单栏
- **且** 用户仍可通过 URL 直接访问该菜单（若路由仍可访问且菜单仍启用）
- **且** 其所有后代菜单也被隐藏

#### Scenario: 显示菜单

- **当** 用户将菜单的 visible 字段设置为显示时
- **则** 该菜单及其所有后代菜单同步显示
- **且** 在权限与状态条件满足时，对应菜单可重新出现在前端菜单栏
