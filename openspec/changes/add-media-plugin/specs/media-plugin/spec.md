## ADDED Requirements

### Requirement: 媒体策略管理

系统 SHALL 提供媒体策略的分页查询、详情、新增、修改、删除、启用状态切换和全局策略设置能力。

#### Scenario: 创建媒体策略

- **WHEN** 用户提交策略名称、YAML 策略内容和启用状态
- **THEN** 系统 SHALL 创建一条媒体策略记录并返回新策略 ID

#### Scenario: 设置全局策略

- **WHEN** 用户将某条策略设置为全局策略
- **THEN** 系统 SHALL 将其他策略的全局标记关闭，并保证最多只有一条策略处于全局状态

#### Scenario: 删除已绑定策略

- **WHEN** 用户删除一个已被设备、租户或租户设备绑定引用的策略
- **THEN** 系统 SHALL 拒绝删除并返回业务错误

### Requirement: 媒体策略绑定管理

系统 SHALL 将设备策略绑定、租户策略绑定和租户设备策略绑定作为三个相互独立的资源管理，分别提供查询、保存和删除接口，不得通过同一个绑定接口和 `scope` 参数混合处理。

#### Scenario: 查询设备策略绑定

- **WHEN** 用户查询设备策略绑定列表
- **THEN** 系统 SHALL 只返回设备国标 ID 与策略的绑定关系

#### Scenario: 保存设备策略绑定

- **WHEN** 用户通过设备策略绑定接口提交设备国标 ID 和策略 ID
- **THEN** 系统 SHALL 为该设备保存唯一策略绑定，重复保存同一设备时更新策略 ID

#### Scenario: 查询租户策略绑定

- **WHEN** 用户查询租户策略绑定列表
- **THEN** 系统 SHALL 只返回租户 ID 与策略的绑定关系

#### Scenario: 保存租户策略绑定

- **WHEN** 用户通过租户策略绑定接口提交租户 ID 和策略 ID
- **THEN** 系统 SHALL 为该租户保存唯一策略绑定，重复保存同一租户时更新策略 ID

#### Scenario: 查询租户设备策略绑定

- **WHEN** 用户查询租户设备策略绑定列表
- **THEN** 系统 SHALL 只返回租户 ID、设备国标 ID 与策略的绑定关系

#### Scenario: 保存租户设备策略绑定

- **WHEN** 用户通过租户设备策略绑定接口提交租户 ID、设备国标 ID 和策略 ID
- **THEN** 系统 SHALL 为该租户下的设备保存唯一策略绑定，重复保存同一租户设备时更新策略 ID

### Requirement: 媒体策略解析

系统 SHALL 按租户设备策略、设备策略、租户策略、全局策略的优先级解析一个设备当前生效策略。

#### Scenario: 解析具体设备策略

- **WHEN** 用户提交租户 ID 和设备国标 ID 查询生效策略
- **THEN** 系统 SHALL 返回匹配到的最高优先级策略及其来源

#### Scenario: 无策略匹配

- **WHEN** 用户提交的租户 ID 和设备国标 ID 无任何绑定且不存在全局策略
- **THEN** 系统 SHALL 返回空策略结果而不是报错

### Requirement: 流别名管理

系统 SHALL 提供流别名分页查询、详情、新增、修改和删除能力。

#### Scenario: 创建流别名

- **WHEN** 用户提交别名、真实流路径和自动移除标记
- **THEN** 系统 SHALL 创建一条流别名记录，并保证别名全局唯一

#### Scenario: 更新流别名

- **WHEN** 用户修改流别名的真实流路径或自动移除标记
- **THEN** 系统 SHALL 保存更新后的流别名配置

### Requirement: 全平台共享媒体配置

系统 SHALL 将媒体策略、策略绑定和流别名作为全平台共享配置管理，不按 LinaPro 宿主租户隔离。

#### Scenario: 保存平台级媒体配置

- **WHEN** 任意有媒体管理权限的用户新增或修改媒体策略、策略绑定或流别名
- **THEN** 系统 SHALL 将该配置保存为平台共享数据，表结构和查询条件都不包含宿主租户隔离字段

### Requirement: 中文-only 插件界面

系统 SHALL 以中文直接呈现 `media` 插件菜单、页面、按钮、表单、错误和接口说明，不维护本模块 i18n 资源。

#### Scenario: 插件页面展示中文文案

- **WHEN** 用户打开媒体插件页面
- **THEN** 系统 SHALL 展示中文菜单、标签、按钮和提示文案，且不依赖 `manifest/i18n` 资源
