## 1. P0 源码升级治理契约

- [ ] 1.1 梳理并落库/落文档稳定扩展面、限定兼容面与内部实现面的分类清单，覆盖宿主、前端工作台与插件开发入口。
- [ ] 1.2 定义框架版本分级、受支持升级路径、breaking changes 标记规则与发布时必须同步交付的升级资料模板。
- [ ] 1.3 补齐源码升级文档基线，新增或更新升级指南、兼容性说明、破坏性变更清单和版本发布说明模板，并同步中英文镜像文档。

## 2. P0 源码升级前检查与演练

- [ ] 2.1 设计并实现源码升级前检查入口，至少校验当前框架版本、受支持升级路径、对非稳定扩展面的高风险定制与必要发布资料是否齐备。
- [ ] 2.2 为二开项目整理标准源码升级流程，给出从旧版本基线升级到新版本的检查、合并、验证与回退建议。
- [ ] 2.3 增加源码升级治理验证，确保升级前检查结果、文档基线和发布清单能够被自动化校验。

## 3. P1 运行时升级元数据与资产约定

- [ ] 3.1 定义业务系统运行时升级资产目录、版本元数据结构和与业务版本绑定的 SQL 文件命名/排序规则。
- [ ] 3.2 新增业务系统版本与迁移记录的宿主表结构、SQL 交付文件及对应 DAO/DO/Entity 工件生成流程。
- [ ] 3.3 实现升级资产解析与校验能力，覆盖版本兼容性、文件顺序、校验值和 SQL 幂等要求。

## 4. P1 运行时升级执行编排

- [ ] 4.1 新增正式升级命令或等价入口，明确与 `init` / `mock` 的职责隔离，并在执行前完成版本与状态预检查。
- [ ] 4.2 实现运行时升级的互斥执行、步骤级记录、失败即停与恢复/续跑机制。
- [ ] 4.3 提供当前业务版本、目标版本、升级状态和历史步骤的查询/诊断能力，便于运维审计与故障排查。

## 5. 验证与发布收口

- [ ] 5.1 补充升级相关自动化测试，覆盖 bootstrap 命令隔离、升级账本记录、失败恢复和受支持升级路径校验。
- [ ] 5.2 建立“新装初始化 + 增量升级 + 失败恢复”验证基线，并纳入后续发布前检查流程。
- [ ] 5.3 输出面向开源用户和业务项目维护者的最终升级说明，明确推荐实践、限制条件与后续版本维护要求。
