## MODIFIED Requirements

### Requirement: 业务系统运行时升级能力在当前变更中仅保留方向性约束
框架 SHALL 在当前变更中仅保留业务系统运行时升级的方向性约束：未来需要提供业务版本号、框架版本号、升级 SQL 绑定和执行记录机制；但本轮实现 MUST 以源码升级（P0）为优先，不在当前迭代中落地业务系统运行时升级功能。

#### Scenario: 当前迭代评估实现边界
- **WHEN** 团队执行本次 `upgrade-governance` 迭代
- **THEN** 业务系统运行时升级能力只作为后续 P1 方向保留
- **AND** 本轮代码实现范围以框架源码升级能力为准
