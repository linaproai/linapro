## 设计

### 页面交互

操作日志和登录日志页面不再渲染表格复选框，也不再维护选中行状态。工具栏保留“清空”作为全量清理入口；“删除”按钮改为日志清理入口，点击后弹出删除对话框。对话框默认按日期范围删除，用户选择开始日期和结束日期后确认执行清理；同时提供“删除所有日志”选择，勾选后忽略日期范围并执行当前权限范围内的全量日志清理。

范围删除复用插件已有 `DELETE /operlog/clean` 和 `DELETE /loginlog/clean` 接口，前端只传递 `beginTime` 与 `endTime`。范围为空且未勾选“删除所有日志”时前端阻止提交并提示选择范围，避免把范围删除误执行为全量清理。选择“删除所有日志”时前端调用同一 `clean` 接口但不传日期参数，由后端已有权限/租户过滤控制清理边界。

### 宿主边界

本变更属于 `linapro-monitor-operlog` 和 `linapro-monitor-loginlog` 源码插件的工作台页面能力补齐，不改变 `apps/lina-core` 核心宿主的领域契约、通用服务语义或存储模型。

### 影响分析

- i18n：新增两个启用 `i18n` 的插件运行时 UI 文案，需同步维护 `zh-CN` 与 `en-US` 插件语言包。
- 数据权限：删除动作继续复用已有插件 `Clean` 服务路径；该路径在数据库操作前应用租户过滤，范围条件在数据库侧生效，不在内存中过滤。
- API 契约：不新增 HTTP API、路由、方法、DTO 或响应字段；复用已有 `DELETE /clean` 范围清理契约。
- 后端 Go：不修改 Go 生产代码、Controller、Service、DAO、运行期依赖或 DI。
- 数据库：不修改 SQL、索引、DAO 生成、软删除或时间字段；日志清理沿用现有硬删除语义。
- 插件：变更限制在两个源码插件自身的前端页面、语言包、E2E 和 OpenSpec 记录，不修改宿主插件生命周期资源归属。
- 缓存一致性：不涉及缓存、快照、失效或集群一致性。
- 开发工具跨平台：不修改 Makefile、脚本、CI、代码生成或 `linactl`。
- 测试：更新两个插件既有 E2E，覆盖表格复选框移除、删除按钮范围选择、全量删除选择、请求参数、弹窗间距和翻译文案。
