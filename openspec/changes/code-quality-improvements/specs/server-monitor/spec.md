## ADDED Requirements

### Requirement: 服务监控前端必须支持可见性感知的自动刷新
服务监控前端页面 SHALL 在挂载后启动周期性轮询刷新最新指标，刷新周期默认 `30s`。轮询 MUST 监听页面可见性事件：标签页隐藏（`document.visibilityState === 'hidden'`）时 MUST 暂停轮询；页面再次可见时 MUST 立即触发一次刷新并恢复周期性轮询。组件卸载时 MUST 显式停止轮询，避免内存泄漏。

#### Scenario: 页面可见时按周期刷新
- **WHEN** 用户停留在服务监控页面
- **AND** 标签页处于可见状态
- **THEN** 前端 MUST 每 30 秒重新请求最新监控指标
- **AND** UI 响应式更新展示

#### Scenario: 标签页隐藏时暂停轮询
- **WHEN** 用户切换到其他标签页
- **THEN** 服务监控页面 MUST 停止周期性请求
- **AND** 不再产生新的网络请求

#### Scenario: 标签页重新可见时立即刷新
- **WHEN** 用户从其他标签页切回服务监控页面
- **THEN** 前端 MUST 立即触发一次刷新
- **AND** 恢复周期性轮询

#### Scenario: 组件卸载时停止轮询
- **WHEN** 用户离开服务监控页面（路由切换或组件卸载）
- **THEN** 前端 MUST 显式停止轮询定时器，不再发起请求
