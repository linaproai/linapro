## MODIFIED Requirements

### Requirement: 系统接口文档页面展示

系统 SHALL 提供一个“系统接口”页面，通过集成 OpenAPI 文档 UI 展示后端 API 文档。文档数据来源于宿主服务自建的 `/api.json` OpenAPI v3 规范文件，而不是直接使用 GoFrame 默认自动产物。

#### Scenario: 正常加载接口文档

- **WHEN** 用户点击“系统信息 > 系统接口”菜单
- **THEN** 页面展示宿主自建 `/api.json` 渲染的 API 文档
- **AND** 文档包含宿主静态接口与当前启用插件的接口定义

### Requirement: 系统接口自动合并动态插件公开路由文档

系统 SHALL 将当前已启用动态插件的路由合同投影到宿主自建 `OpenAPI` 文档中，并展示动态插件对外真实可访问的固定公开路径。

#### Scenario: 动态插件禁用后从系统接口中移除

- **WHEN** 一个已启用的动态插件被禁用、卸载或切换到不再暴露该路由的激活版本
- **THEN** 宿主从自建 `OpenAPI` 文档中移除该插件对应的动态路由投影
- **AND** “系统接口”页面不再展示已失效的公开路径

## ADDED Requirements

### Requirement: 系统接口按源码插件启用状态投影路由文档

系统 SHALL 仅将当前已启用源码插件的已采集路由绑定投影到宿主 `OpenAPI` 文档中，而不根据 URL 前缀推断源码插件路由归属。

#### Scenario: 已启用源码插件接口出现在系统接口

- **WHEN** 一个源码插件已经启用，且宿主已经在其注册阶段采集到标准 DTO 路由绑定
- **THEN** 用户访问“系统接口”页面时能够看到该源码插件对应的接口路径
- **AND** 这些接口保留源码插件在 DTO `g.Meta` 中声明的方法、标签、摘要、描述与权限信息

#### Scenario: 已禁用源码插件接口从系统接口中移除

- **WHEN** 一个源码插件被禁用
- **THEN** 宿主从自建 `OpenAPI` 文档中移除该源码插件对应的路由投影
- **AND** 即使真实路由仍保留在宿主路由表中，系统接口页面也不会继续展示该源码插件接口

### Requirement: 系统接口文档排除非业务与内部治理路由

宿主 SHALL 在构建自建 `OpenAPI` 文档时排除静态兜底、文件兜底、自建 `/api.json` handler 与动态插件固定分发入口等非业务路由。

#### Scenario: 过滤内部技术路由

- **WHEN** 宿主扫描真实路由表生成系统接口文档
- **THEN** 宿主不会把静态前端兜底路由、动态插件固定分发入口、自建 `/api.json` 输出路由或其他内部技术路由加入系统接口文档
- **AND** 系统接口页面只展示对业务调用方有意义的接口
