## ADDED Requirements

### Requirement: 系统信息页面必须按当前语言展示项目介绍与组件说明
系统 SHALL 在系统信息页面和系统信息 API 中按当前请求语言返回项目描述、组件说明等展示性文案。系统信息国际化 MUST 保持项目定位和组件标识稳定，只对面向用户的描述性文本做本地化投影。

#### Scenario: 英文环境下展示系统信息
- **WHEN** 用户以 `en-US` 打开系统信息页面或请求系统信息 API
- **THEN** 关于项目区块中的项目描述使用英文本地化结果
- **AND** 前后端组件说明使用英文本地化结果
- **AND** 组件名称、版本号和链接保持原值

#### Scenario: 组件说明缺失时回退默认语言
- **WHEN** 某个组件在当前语言下缺少说明文案
- **THEN** 系统回退显示默认语言说明
- **AND** 该组件仍然正常显示在对应区块中

### Requirement: 系统信息国际化必须覆盖公共项目定位文案
系统 SHALL 让项目名称、项目介绍和框架定位描述在多语言场景下保持语义一致，确保 `LinaPro` 始终被描述为“AI驱动的全栈开发框架”而不是在不同语言下出现定位漂移。

#### Scenario: 不同语言下保持统一项目定位
- **WHEN** 用户切换系统语言查看系统信息页
- **THEN** 页面中的项目定位文案只在语言表达上变化
- **AND** 不会把 LinaPro 描述成单一后台管理系统或其他偏离框架定位的产品
