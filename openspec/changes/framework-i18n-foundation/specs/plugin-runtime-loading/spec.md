## MODIFIED Requirements

### Requirement: 动态插件运行时产物携带可治理的路由合同

系统 SHALL 允许动态插件在运行时产物中携带后端动态路由合同，宿主装载产物后能够恢复这些路由的路径、方法与最小治理元数据，而不需要在请求时再次扫描源码目录。动态路由合同中由插件或插件中间件消费的自定义路由声明 SHALL 通过通用 `meta` 字段透传；宿主 SHALL NOT 为某个业务插件定义或校验专用路由字段。

#### Scenario: 构建阶段提取动态路由合同

- **WHEN** 构建动态插件运行时产物
- **THEN** 构建器从 `backend/api/**/*.go` 中的请求结构体 `g.Meta` 提取动态路由元数据
- **AND** 将路径、方法、文档与宿主治理字段写入运行时产物中的专用区段
- **AND** 将不属于宿主路由合同字段的插件自定义声明写入路由合同 `meta`
- **AND** 宿主加载产物后可恢复为动态插件 `manifest.Routes`

#### Scenario: 宿主校验动态路由合同

- **WHEN** 宿主装载一个动态插件的路由合同
- **THEN** 宿主校验内部路径、方法、`access` 与 `permission` 是否合法
- **AND** `access` 未声明时按 `login` 处理
- **AND** `public` 路由不得声明 `permission`
- **AND** 宿主不校验或解释 `meta` 中的插件自定义路由声明
- **AND** 非法合同会导致该产物装载失败
