## MODIFIED Requirements

### Requirement:插件管理页面聚焦关键治理信息

系统 SHALL 优先展示插件管理页面首屏所需的关键治理字段，避免在列表概览中挤满低价值信息。系统 SHALL 优化插件管理首次进入时的前端路由加载成本和后端列表构建成本；列表接口 MUST 返回分页、有上限的摘要读模型，详情、安装、卸载、升级和授权确认弹窗所需的完整治理字段 MUST 在用户打开对应工作流时按需加载。

#### Scenario:显示源码插件的治理类别
- **WHEN** 清单中的插件类型为 `source`
- **THEN** 插件管理页面在"插件类型"列显示为"源码插件"

#### Scenario:显示动态插件的治理类别
- **WHEN** 清单中的插件类型为 `dynamic`
- **THEN** 插件管理页面在"插件类型"列一致显示为"动态插件"
- **AND** 当前 `dynamic` 插件在治理列表中仍显示和筛选为"动态插件"，即使具体产物是 `wasm` 产品

#### Scenario:简化插件管理列表概览字段
- **WHEN** 管理员查看插件管理列表
- **THEN** 页面不再显示"交付模式"、"集成模式"和"入口"列
- **AND** 页面在"更新时间"前显示"安装时间"

#### Scenario:首次进入加载分页摘要列表
- **WHEN** 管理员首次进入插件管理页面
- **THEN** 页面仅请求 `GET /plugins` 分页摘要列表渲染首屏表格
- **AND** 列表响应包含表格、筛选结果和行级操作判断所需字段
- **AND** 列表响应不包含 `dependencyCheck`、`requestedHostServices`、`authorizedHostServices`、`declaredRoutes`或 cron 声明详情
- **AND** 页面不得为了渲染首屏列表对每一行自动请求 `GET /plugins/{id}`

#### Scenario:打开详情时按需加载完整治理字段
- **WHEN** 管理员从列表点击插件详情操作
- **THEN** 页面打开详情弹窗前请求 `GET /plugins/{id}`
- **AND** 详情弹窗使用服务端返回的完整治理字段展示依赖检查、宿主服务授权、动态路由和必要的 cron 审查信息
- **AND** 前端不得根据首屏摘要自行推断依赖图、授权范围或动态路由审查语义

#### Scenario:打开治理动作弹窗时按需加载动作数据
- **WHEN** 管理员打开安装授权、卸载确认或升级确认工作流
- **THEN** 页面按该工作流请求详情、依赖检查或升级预览等服务端接口
- **AND** 弹窗展示的阻断原因、授权范围、下游依赖和升级差异来自服务端结果
- **AND** 首屏列表返回的数据不得作为这些动作的唯一决策依据

#### Scenario:首屏路由不同步加载重型弹窗组件
- **WHEN** 管理员首次进入插件管理页面且尚未打开任何弹窗
- **THEN** 前端首屏路由 chunk 只包含表格、筛选、行操作和必要轻量 helper
- **AND** 详情、动态上传、安装授权、卸载、升级和生命周期前置条件弹窗组件按需异步加载
- **AND** 弹窗按需加载不得改变既有权限控制、确认交互或错误提示语义

#### Scenario:摘要预热失败时请求回退同步构建
- **WHEN** 宿主启动后的插件管理摘要读模型预热失败或尚未完成
- **AND** 管理员进入插件管理页面
- **THEN** 列表接口同步构建分页摘要读模型并返回
- **AND** 构建成功后后续列表查询可复用该摘要读模型
- **AND** 请求回退不得同步构建详情、安装、卸载、升级和授权确认弹窗专用字段
