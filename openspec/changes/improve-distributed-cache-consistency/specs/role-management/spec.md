## ADDED Requirements

### Requirement: 权限拓扑缓存必须跨节点可靠失效

系统 SHALL 在角色、菜单、用户角色、插件权限菜单或权限资源关系变化后，通过统一缓存协调机制可靠失效所有节点的 token 权限快照。

#### Scenario: 角色菜单权限变更
- **WHEN** 管理员更新角色菜单或按钮权限
- **THEN** 系统提交角色权限关系变更
- **AND** 系统可靠发布权限拓扑缓存修订号
- **AND** 所有节点在权限缓存域允许的陈旧窗口内丢弃旧 token 权限快照

#### Scenario: 菜单权限标识变更
- **WHEN** 管理员新增、修改、删除或禁用菜单权限
- **THEN** 系统发布权限拓扑缓存修订号
- **AND** 后续受保护 API 权限校验不得长期使用旧菜单权限拓扑

#### Scenario: 插件权限拓扑变更
- **WHEN** 插件安装、启用、禁用、卸载或同步导致插件菜单或按钮权限变化
- **THEN** 系统发布权限拓扑缓存修订号
- **AND** 受影响权限在所有节点上按最新插件状态参与授权判断

### Requirement: 权限拓扑失效发布失败不得静默成功

权限拓扑关键写路径 MUST 在无法发布权限缓存修订号时返回结构化错误或回滚业务变更，避免集群节点继续使用旧授权快照。

#### Scenario: 权限修订号发布失败
- **WHEN** 角色、菜单或用户角色写路径需要发布权限拓扑修订号但发布失败
- **THEN** 系统返回结构化业务错误
- **AND** 调用端不得收到权限变更已经完全生效的成功响应
- **AND** 系统记录失败原因用于重试或修复

#### Scenario: 受保护 API 遇到过期权限缓存
- **WHEN** 受保护 API 校验权限时无法确认本地权限快照 freshness 且超过故障窗口
- **THEN** 系统按失败关闭策略拒绝该请求
- **AND** 系统不得因为旧本地权限快照继续放行不确定权限
